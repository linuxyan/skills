## Phase 5：输出至 IMA 笔记本

> **目标**：每次生成估值报告后，自动将 Markdown 报告写入 IMA "股票估值"笔记本。

### 5.1 笔记本定位

```
笔记本名称：股票估值
笔记本 folder_id：folder18c42fb8e51fe8c3
```

> 如需验证笔记本是否存在，可调用 IMA API。若 folder_id 错误，`import_doc` 会返回 `{code:310001}`（笔记本不存在）。

### 5.2 写入流程

```
生成 Markdown 报告（Phase 4）
    ↓
UTF-8 校验（必须，防止乱码）
    ↓
直接写入 IMA notebook（import_doc API）
    ↓
记录 note_id 到报告头部注释
```

### 5.3 完整写入脚本

> ⚠️ **重要**：不要通过 shell 调用 `ima_api.cjs`（shell 会将 Markdown 中的括号误判为子shell）。必须将内容写入临时 JS 文件，再用 Node.js 直接执行。

**写入脚本模板**（保存为 `/tmp/ima_write_report.js`）：

```javascript
#!/usr/bin/env node
/**
 * 写入估值报告到 IMA 笔记本
 * 用法：node /tmp/ima_write_report.js "报告内容（Markdown）" "笔记本folder_id"
 */
const fs = require('fs');

const clientId = fs.readFileSync('/Users/yan/.config/ima/client_id', 'utf8').trim();
const apiKey = fs.readFileSync('/Users/yan/.config/ima/api_key', 'utf8').trim();
const content = process.argv[2] || fs.readFileSync('/tmp/report_body.md', 'utf8');
const folderId = process.argv[3] || 'folder18c42fb8e51fe8c3';

const body = { content_format: 1, content, folder_id: folderId };

async function apiCall(path, body) {
  const res = await fetch(`https://ima.qq.com/${path}`, {
    method: 'POST',
    headers: {
      'ima-openapi-clientid': clientId,
      'ima-openapi-apikey': apiKey,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(body),
  });
  const text = await res.text();
  const json = JSON.parse(text);
  if (json.code !== 0) throw new Error(`IMA API error: ${json.code} ${json.msg}`);
  return json.data.note_id;
}

async function main() {
  const noteId = await apiCall('openapi/note/v1/import_doc', body);
  console.log('✅ 报告已写入 IMA，note_id:', noteId);
}

main().catch(e => { console.error('❌', e.message); process.exit(1); });
```

### 5.4 执行方式（两步走）

**Step 1**：将报告内容保存到临时文件
```bash
cp "/path/to/{股票名称}_估值报告.md" /tmp/report_body.md
```

**Step 2**：执行写入脚本
```bash
node /tmp/ima_write_report.js "$(cat /tmp/report_body.md)" "folder18c42fb8e51fe8c3"
```

### 5.5 UTF-8 校验（强制）

写入前必须校验报告内容为合法 UTF-8：
```bash
node -e "const b=require('fs').readFileSync('/tmp/report_body.md');try{Buffer.from(b,'utf8');console.log('UTF-8 OK')}catch(e){console.error('❌ 非UTF-8:',e.message)}"
```

### 5.6 错误处理

| IMA 返回码 | 含义 | 处理方式 |
|-----------|------|---------|
| `code: 0` | 成功 | 记录 note_id，完成 |
| `code: 310001` | 笔记本不存在 | 告知用户检查"股票估值"笔记本 |
| `code: 100001` | 参数错误 | 检查 content_format 和 folder_id |
| `code: 100009` | 内容超限 | 拆分报告，用 `append_doc` 分批写入 |
| 其他非0码 | API 错误 | 读取 `msg` 字段，向用户展示 |

### 5.7 报告头部注释规范

写入 IMA 时，在报告 Markdown 开头追加 IMA 元信息注释：

```markdown
<!--
IMA: note_id=7457971323818554, folder=folder18c42fb8e51fe8c3, 写入时间=2026-05-07
-->
```

> - 若该股票已有笔记，用 `append_doc` 追加到同一篇
> - 或直接新建一篇（保留历史版本），在标题中注明版本日期
