# Phase 1 数据查询详细规则

本文档包含 Phase 1 获取财务数据的详细规则，包括各数据项的取值说明、分红双次检测流程、编码处理和财务洗澡检测机制。

---

## 各数据项详细说明

### ① 最新年报 EPS

- **取值规则**：必须使用最新年报的**基本每股收益**（非稀释每股收益）
- **优先级**：年报 EPS > 业绩预告 EPS > TTM EPS（降级使用需注明）
- **解读要点**：注意确认 EPS 对应的年度
- **验证方式**：可交叉验证 `归母净利润 ÷ 总股本 ≈ EPS`

### ② 最新股价 + 行情指标

- **取值**：最新交易日的收盘价
- **配套查询**：同时获取 PE(TTM)、PB、股息率、年初至今涨跌幅等
- **注意**：非交易时段使用最近一个交易日收盘价

### ③ 近 5 年加权 ROE

- **取值**：**加权平均净资产收益率**（非全面摊薄 ROE）
- **计算方式**：取近 5 个年度的加权 ROE，计算算术平均值
- **注意**：如果最新年报 ROE 异常低（如因一次性亏损），可考虑是否排除，但必须在报告中说明理由
- **API 字段**：`fina_indicator` 中的 `roe` 字段

### ④ 近 5 年分红数据

- **必须获取的信息**：每年全年每股股利（元）、是否含中期分红、全年分红率
- **计算全年分红率**：`全年每股股利 ÷ 当年 EPS × 100%`

---

## ⛔ 强制规则：每次估值必须先确认"是否双次分红"

A 股分红分为**中期分红**（H1，约 7-9 月公告）和**年度分红**（次年 4-6 月公告），两者独立。API 返回的分红记录**按公告日期排列**，不按归属年度汇总——这是最常见的数据陷阱。

**Step 4-A：先查询全年分红概览（必做）**

```
"{公司名}{代码} 2020-2025历年分红送配每股派息分红率年度分红中期分红"
```

**Step 4-B：扫描是否双次分红（必做，不得跳过）**

| 判断 | 后续动作 |
|------|---------|
| 同一年度有两条分红记录 | ✅ 确认双次分红 → 转到 Step 4-C |
| 同一年度只有一条，且为"年度分红"/"年报分红" | 可能遗漏中报 → 转到 Step 4-D 单独验证 |
| 同一年度只有一条，且公告日期在 7-9 月 | 大概率只是中报 → 转到 Step 4-D 补查年报分红 |

**Step 4-C：按归属年度汇总双次分红**

```
全年每股股利 = 中报每股派息 + 年报每股派息
全年分红率 = 全年每股股利 ÷ 当年 EPS × 100%
```

**Step 4-D：单独验证（对疑似只有一次的年度）**

如果 Step 4-B 不确定，**必须**分别查询中报和年报：
```
关键词1: "{公司名}{代码} {年份}年中期分红方案每股派息"
关键词2: "{公司名}{代码} {年份}年年报分红方案每股派息"
```

**Step 4-E：最终校验（必做）**

- 如果分红率 < 20% 且该股票历史上分红率稳定在 30%+ → ⚠️ 极可能遗漏了中报分红
- 如果全年每股股利明显低于预期 → 重新检查是否有中报分红

### ⛔ 已发生的错误案例（必须引以为戒）

| 案例 | 错误 | 正确数据 | 教训 |
|------|------|---------|------|
| **东阿阿胶**（2026-05-07） | 遗漏年报分红10派14.31（乱码未识别），误将34.16%当作分红率 | 分红率从34%修正为≈100% | ①乱码下"10派X.XX"数字可见但被忽略 ②误读34.16%标签 |
| **招商银行**（2026-05-07） | 仅取中报10派10.13元，遗漏年报10派10.03元 | 分红率从17.7%修正为35.4% | 即使API返回了合计值，也必须逐条拆解验证 |
| **洋河股份** | "中期不分红"被误读为全年不分红 | 年度分红是独立的 | "中期不分配"≠"全年不分配" |

**记忆口诀**：**每次估值，先问自己"这家公司一年分几次？"**——宁可多查一次确认，也不能漏掉。

- **分红率 > 100% 的处理**：说明公司动用了留存利润分红，需在报告中注明原因

### ⑤ 分红政策/承诺/规划

- **查询要点**：同时搜索当前有效规划和历史规划变更记录
- **确定优先级（从高到低）**：
  1. **公司最新分红规划**：优先采用，反映管理层未来意图
  2. **⚠ 必须确认承诺是否仍然有效**：搜索"分红规划 变更/修订/取消/调整"
  3. **绝对不能仅凭旧公告就采信**（如洋河 2026 年取消了"不低于70亿元"的硬承诺）
  4. **近 3-5 年实际分红率均值**：在没有明确承诺时使用
  5. **注意区分"年度分红"和"特别分红/中期分红"**：一年分两次的必须加总
- **报告中的标注**：如有承诺→说明具体内容和期限；如承诺已变更→⚠ 标注变更时间和差异；如无承诺→说明情况

### ⑥ 近 5 年经营现金流净额

- **用途**：Phase 3 筛选条件"现金流健康"检验
- **检验标准**：近 3 年平均经营现金流 ≥ 净利润（比值 ≥ 100%）
- **解读要点**：单年低于 100% 不一定有问题，但**连续多年低于 100% 是警告信号**
- **经营现金流为负是重大风险**，说明分红可能面临执行困难
- 计算"净利润现金含量"= `经营现金流净额 ÷ 归母净利润 × 100%`

### ⑦ 资产负债率 + 有息负债率

- **必须同时查询两个指标**：
  - **资产负债率** = 总负债 ÷ 总资产（一般标准：< 70%）
  - **有息负债率** = (短期借款 + 一年内到期的非流动负债 + 长期借款 + 应付债券) ÷ 总资产
- **判断"借钱分红"嫌疑**：资产负债率高 + 有息负债率高 + 分红率高 → ⚠️
- **辅助判断**：财务费用为负（利息收入 > 支出）说明公司资金充裕

### ⑧ 近 5 年每年 EPS（用于 g_final 可信度评估）

- **用途**：计算 EPS 复合增速 CAGR，与 g_final 对比
- **计算公式**：`CAGR = (EPS_最新年 / EPS_最早年)^(1/年数) - 1`
- **解读**：CAGR ≥ g_final → 历史增速支撑；CAGR < g_final 的 50% 或持续下滑 → 增速假设不可靠
- 此数据可与 ① 合并查询

### ⑨ 行业趋势信息

- **查询方式**：搜索该行业最新趋势分析
- **需了解**：行业所处阶段（增长/存量/衰退）、产量/规模增速、重大政策变化、主要竞争动态
- **评分参考**：有增长空间→高分，接近饱和但稳定→中分，衰退/调整→低分

### ⑩ 最新季度业绩（年报已发布时跳过）

- **用途**：年报尚未发布时展示最新季度/前三季/预告数据
- 如已有最新年报数据，此项不需要查询

### ⑪ 年度分红方案（年报刚发布时补充查询）

- 如果查不到年度分红方案，在报告中注明"年度分红方案待股东大会确认"，用承诺值或历史均值估算

### B④ 预收款项/合同负债（TOP/STRONG档用）

- 此项专为 Phase 2.1 顶级品牌公司增强而设
- 当公司为高ROE高分红消费品牌（疑似TOP/STRONG档）时需查询，普通股票无需查询

## ⛔ 编码乱码处理详细规则

**背景**：NeoData API 返回的中文可能因终端编码问题显示为乱码。`query.py` 已升级为终端输出使用 `ensure_ascii=True`（中文转义为 `\uXXXX`），不再产生乱码。

**推荐做法**：查询时始终使用 `--output-file` 参数保存结果到文件，再用 Read 工具读取——文件中的 UTF-8 中文完全正确。

```
Step E1：执行查询（推荐方式）
        python3 scripts/query.py --query "xxx" --output-file /tmp/result.json --pretty
        
Step E2：读取结果
        使用 Read 工具读取 /tmp/result.json → 中文完全正确，无乱码风险

Step E3：备用方案（直接终端输出）
        如未使用 --output-file，终端输出为 \uXXXX 转义格式
        → 数字和英文关键字段可直接提取
        → Unicode 转义序列是可解析的（如 \u4e1c\u963f = "东阿"）
```

**查询调用模板**：

```bash
# 1. 基础数据查询
python3 scripts/query.py --query "{公司名}{代码} 最新股价 PE PB ROE EPS" \
  --output-file /tmp/{代码}_basic.json --pretty

# 2. 分红数据查询
python3 scripts/query.py --query "{公司名}{代码} 2020-2025 历年分红送配" \
  --output-file /tmp/{代码}_dividend.json --pretty

# 3. 读取结果：使用 Read 工具读取上述 JSON 文件
```

## ①② 财务洗澡检测 + 偏差熔断机制（⚠ 必须执行，自动触发）

**目的**：在估值计算前，自动检测EPS是否因会计政策选择而失真，触发熔断时强制输出双版本报告。

### 第一步：自动化偏差计算（每次估值必须执行）

**计算还原EPS（OCF法）：**
```
还原EPS = 近3年平均经营现金流净额 ÷ 当期末总股本
```

**计算偏差率：**
```
偏差率 = |报表EPS - 还原EPS| / 报表EPS × 100%
```

**偏差熔断阈值：**

| 偏差率 | 熔断等级 | 触发动作 |
|--------|---------|---------|
| ≤ 15% | ✅ 不触发 | 正常计算，使用报表EPS |
| 15% < 偏差率 ≤ 30% | ⚠️ 观察级 | 正常计算，列示还原EPS供参考，注明"⚠️ EPS存在轻微失真风险" |
| > 30% | 🔴 熔断级 | **强制输出双版本报告** |

### 第二步：红旗信号自动扫描

**A. 经营现金流匹配度异常（自动扫描近5年）**

| 红旗信号 | 含义 | 附加动作 |
|---------|------|---------|
| 连续2年现金含量 < 0.7 | 利润质量差 | 附加⚠️标注 |
| 单年现金含量暴降至 < 0.3，次年恢复 | 疑似洗澡年 | 剔除洗澡年，用近3年均值重新计算还原EPS |
| 单年EPS同比下滑 > 50% 但现金流正常 | 典型洗澡特征 | 自动标注"疑似一次性洗澡" |

**B. 会计差错更正/追溯调整公告（主动查询）**

查询：`"{公司名}{代码} 会计差错更正 追溯调整 前期会计差错"`

| 情形 | 附加动作 |
|------|---------|
| 调整幅度 ≤ 30% | 在报告中注明，不触发熔断 |
| **调整幅度 > 30%** | **自动触发🔴熔断** |

**C. 董事长/CFO更换 + 业绩暴雷（主动查询）**

查询：`"{公司名}{代码} 董事长 更换 辞职 被查"`、`"{公司名} CFO 财务总监 更换"`

| 情形 | 附加动作 |
|------|---------|
| 新董事长上任后首份年报业绩暴雷 | 用上任前3年EPS均值替代洗澡后EPS |
| CFO频繁更换（3年换2次以上） | 标记"⚠️ 财务政策不连续" |
| 董事长被查/留置后业绩大幅下调 | **自动触发🔴熔断** |

### 第三步：还原EPS计算（三选一，按优先级）

| 方法 | 公式 | 适用场景 | 优先级 |
|------|------|---------|-------|
| **OCF法** | `还原EPS = 近3年平均经营现金流净额 ÷ 总股本` | 通用，最稳健 | ★★★ |
| **均值法**（剔除异常年） | `还原EPS = 近3年EPS均值（剔除洗澡年）` | 无现金流异常但EPS被操纵 | ★★ |
| **行业对标法** | `还原EPS = 行业平均EPS × 市场份额调整` | 极端异常 | ★ |

### 第四步：熔断输出规则

```
if 偏差率 > 30% or (B类触发且调整幅度>30%) or C类触发:
    输出版本A = 报表EPS计算的安全买入价（标注"报表版"）
    输出版本B = 还原EPS计算的安全买入价（标注"保守版"）
    安全买入价结论 = min(版本A, 版本B)
    报告注明："⚠️ 财务洗澡嫌疑，偏差率{X}%>30%，取较低者¥XX作为绝对安全价"
else if 15% < 偏差率 ≤ 30%:
    仅输出版本A，并列展示还原EPS供参考
else:
    仅输出版本A，无特殊标注
```

### 第五步：报告标注要求

无论是否触发熔断，都必须：
1. 在「核心输入参数」表格中同步列示报表EPS、还原EPS（偏差>15%时）、偏差率
2. 在「估值计算过程」Step 1 中说明财务洗澡检测结果
3. 估值结论中：触发熔断→两版本并列，较低者加粗；未触发→无特殊标注

## 查询执行建议

- **可并行查询**：11 项数据之间无依赖关系，应尽量并行发起
- **关键词灵活调整**：第一次查询不理想时换关键词重试
- **交叉验证**：同一数据项可通过不同查询关键词验证一致性
