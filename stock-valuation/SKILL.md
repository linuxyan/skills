# 内生增长分红估值模型（V1.0）

## 模型简介

本 skill 实现"内生增长分红估值模型"，核心逻辑：
- 以公司的 **ROE**（盈利效率）与 **分红率（Payout）** 推导内生增长率
- 以"十年后分红收益率（Yield on Cost）达到 **8%**"为目标
- 反推**当前安全买入价**

> 参考文档：`references/model_spec.md`

---

## 工作流程

### Phase 1：获取财务数据

使用金融数据 API 获取以下数据：

| 数据项 | 说明 | 推荐 API |
|--------|------|---------|
| EPS（最新年报） | 每股收益 | `fina_indicator` 或 `income` |
| ROE（近5年均值） | 净资产收益率 | `fina_indicator` 字段 `roe` |
| 分红率 | Payout Ratio | `dividend` 或 `fina_indicator` 字段 `or` 推算 |
| 最新股价 | 用于判断是否低于安全价 | `daily`（最新收盘价）|

**数据获取要点（⚠ 关键，严格执行）：**

1. **必须确认最新年报年份**：查询前先确认当前日期，判断最新年报是否已发布（A股年报截止 4/30）。如果当年年报已发布，**必须使用最新年报数据**，不要使用 TTM 或旧年报数据。例如：在 2026 年 5 月做估值，2025 年报（2026-04-30 前发布）应该已经可用，应使用 2025 年报的 EPS 和 ROE。

2. **`ts_code` 必须带交易所后缀**（如 `600519.SH`、`000001.SZ`）

3. **ROE 取近 5 个年度报告期的加权平均净资产收益率**（非简单算术平均更好，但算术平均也可接受）

4. **分红率的确定优先级（从高到低）**：
   - **① 公司最新的分红政策/承诺/规划**（如"2024-2026年分红率不低于75%"）：优先采用承诺值，因为反映管理层未来意图
   - **⚠ 但必须确认承诺是否仍然有效**：搜索时同时查询"分红规划 变更/修订/取消/调整"，确认是否有新规划替代旧规划。分红承诺可能被修改甚至取消（如洋河 2026 年取消了"不低于70亿元"的硬承诺），**绝对不能仅凭旧公告就采信**
   - **② 近 3-5 年实际分红率均值**：在没有明确承诺时使用
   - **③ 注意区分"年度分红"和"特别分红/中期分红"**：有些公司一年分两次，必须加总计算全年分红率
   - 若数据不全，可用近 3 年替代

5. **分红数据解读时必须区分"中期"与"年度"**（⚠ 常见错误，严格执行）：
   - A 股公司分红通常分为**中期分红**（H1，约 8-9 月公告）和**年度分红**（全年，次年 4-6 月公告），两者是独立方案
   - API 返回的"2025年分红送配方案"如果是 8 月份左右公告的，大概率是**2025年中期分红**，而非全年分红
   - **"2025年中期不分红" ≠ "2025年全年不分红"**——必须确认年度分红方案是否也已公告
   - 解读规则：看到"不分配不转增"时，**先确认这是哪个期间（中期还是年度）的方案**，再判断全年分红情况。如果年报尚未发布，年度分红一定尚未公告
   - **错误案例**：洋河 2025-08-19 公告"2025年不分配不转增"实为中期方案（因上半年业绩差），但被误读为"2025年全年不分红"

6. **EPS 必须使用最新年报数据**（非 TTM）。如果最新年报尚未发布，才降级使用 TTM 数据，并在报告中注明

7. **查询时务必检查搜索结果中的数据年份标注**，API 返回的多期数据中可能包含不同年度的报告，需要识别最新一期

8. **解读数据时避免过度推断**：从 API 拿到原始数据后，先理解每条记录的含义和上下文，再得出结论。常见陷阱：
   - 混淆"中期分红"与"年度分红"（见第 5 条）
   - 把"预告"当"实际"（业绩预告区间 ≠ 年报终值）
   - 把"T-1 年度分红"当作"T 年度的承诺"（分红方案对应的是哪个年度的利润）

9. **必须额外查询近 5 年 EPS 数据**（用于 Phase 2.5 的 g_final 可信度评估）：获取 2020-2024（或最新 5 年）每年的 EPS，用于计算 EPS 复合增速 CAGR，并与 g_final 对比

10. **必须额外查询行业趋势信息**（用于 Phase 2.5 的行业空间评估）：了解该公司所在行业的渗透率、增速趋势、政策环境等，判断行业天花板是否限制增长

### Phase 2：执行估值计算

运行估值脚本：

```bash
python3 /Users/yan/.workbuddy/skills/stock-valuation/scripts/valuation.py \
  --eps <EPS_curr> \
  --roe <ROE_5y_avg> \
  --payout <Payout_5y_avg> \
  [--target-yield 0.08] \
  [--years 10] \
  [--stock-name "股票名称"] \
  [--current-price <最新股价>]
```

#### Phase 2.5：g_final 增速可信度评估（⚠ 必须执行）

算出 g_final 后，**必须立即评估该增速在未来 10 年的实现概率**。g_final 是模型的核心假设——如果这个增速根本无法实现，安全买入价就是空中楼阁。

**评估维度（4 项，每项 1-3 分）：**

| 维度 | 评估内容 | 3 分（高可信） | 2 分（中可信） | 1 分（低可信） |
|------|---------|--------------|--------------|--------------|
| **① 历史增速** | 近 5 年 EPS 复合增速 vs g_final | 历史 CAGR ≥ g_final | 历史 CAGR 在 g_final 的 50%-100% | 历史 CAGR < g_final 的 50% 或 EPS 持续下滑 |
| **② 再投资效率** | 留存利润再投资是否真的能创造价值 | 高分红率（≥70%）：不依赖再投资，g_final 天然容易实现 | 中分红率（40%-70%）：适度依赖再投资 | 低分红率（<40%）：高度依赖再投资，需优质投资机会 |
| **③ 行业空间** | 行业渗透率、增速趋势、天花板 | 行业仍有增长空间或稳健需求 | 行业接近饱和但需求稳定 | 行业衰退或深度调整、需求萎缩 |
| **④ ROE 稳定性** | 近 5 年 ROE 波动幅度 | ROE 标准差 < 3个百分点 | ROE 标准差 3-6个百分点 | ROE 标准差 > 6个百分点或单年骤降 > 10个百分点 |

**总分与可信度等级：**

| 总分 | 等级 | 含义 | 报告标记 |
|------|------|------|---------|
| 10-12 | 🟢 **高可信** | g_final 实现概率大，模型结果可靠 | 🟢 高可信 |
| 7-9 | 🟡 **中可信** | g_final 有一定不确定性，需关注风险因素 | 🟡 中可信 |
| 4-6 | 🟠 **低可信** | g_final 实现困难，安全买入价可能偏高 | 🟠 低可信 |

**执行要点：**

1. **数据来源**：在 Phase 1 获取数据时，需额外查询近 5 年 EPS 数据（用于计算 CAGR）和行业趋势信息
2. **评分依据**：评分必须基于已查询到的客观数据，每项评分需在报告中注明具体理由
3. **g_base vs g_final 的差异**：如果 g_final = g_base（未被截断），说明增速来自实际留存再投资能力，可信度天然更高；如果 g_final = Cap（被截断），说明理论增速高于上限，需要更谨慎地评估
4. **特别关注**：当 g_final 被截断时（即 g_base > Cap），需额外说明"g_base 为 X% 但被截断至 Y%，截断幅度越大，说明模型越保守，可信度加分"

| 历史分红稳定 | 近 5 年每年分红率均 > 30% | ✅ / ⚠️ / ❌ |

## 核心公式速查

```
安全买入价 = [ EPS_curr × (1 + g_final)^10 × Payout ] / 8%

g_base   = ROE_5y × (1 - Payout_5y)

g_final 根据 ROE 档位截断：
  ROE > 25%       → min(g_base, 8%)
  15% ≤ ROE ≤ 25% → min(g_base, 5%)
  ROE < 15%       → min(g_base, 3%)
```

## 注意事项

- 目标收益率默认 8%，极其确定的资产（如长电）可调整为 6%-7%（调用脚本时传 `--target-yield 0.06`）
- 模型仅供参考，不构成投资建议
- 数据质量决定结果质量，分红数据缺失时需明确说明
