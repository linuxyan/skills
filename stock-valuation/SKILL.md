# 内生增长分红估值模型（V1.1）

## 模型简介

本 skill 实现"内生增长分红估值模型"，核心逻辑：
- 以公司的 **ROE**（盈利效率）与 **分红率（Payout）** 推导内生增长率
- 以"十年后分红收益率（Yield on Cost）达到 **8%**"为目标
- 反推**当前安全买入价**

> 核心公式速查见本文件底部"核心公式速查"章节

---

#### 1.2 通用规则（⚠ 关键，严格执行）

**A. 年报时效判断**：A 股年报截止 4/30。如果当年年报已发布，**必须使用最新年报数据**；如未发布，用最新可用年报+最新季度数据。

**B. 股票代码格式**：`ts_code` 必须带交易所后缀（上交所 `.SH`，深交所 `.SZ`）。

**C. 数据年份识别**：API 返回多期数据可能混合不同年度，**务必检查每条记录的年份标注**。

**D. 数据解读防错**：
- "预告" ≠ "实际"
- 避免过度推断，先理解上下文和字段含义
- 数据不完整时降级处理，需在报告中注明

**D2. ⛔ 禁止估算，必须实际查询**：每一项数据都必须通过 API 实际查询。数据缺失时在报告中注明并降级评分（对应维度最高只能给到该维度满分的 2/3）。

**E. ⛔ 编码乱码处理**：推荐使用 `--output-file` 保存结果到文件再用 Read 工具读取。记忆口诀：**查询必存文件，Read 工具读取。中文零乱码，数据零误读。**

### Phase 2：执行估值计算

估值计算包含多个子阶段，按顺序执行：

```
Phase 2.1：100%分红率特殊处理（Payout ≥ 95%时触发）
    ↓
Phase 2.4：周期行业适配增强（自动对所有股票执行）
    ↓
Phase 2.5：g_final 增速可信度评估 + 模型适配度评分（必须执行）
    ↓
Phase 2.6：估值风险维度——PE分位数 + 行业结构性断裂识别（必须执行）
    ↓
Phase 2.7：银行股专属修正引擎（仅银行股执行）
    ↓
Phase 3：筛选条件检验
```

#### Phase 2.1：100% 分红率特殊处理（⚠ Payout ≥ 95% 时触发）

核心问题：标准公式 `g_base = ROE × (1 - Payout)` 中，`(1 - Payout)` 趋于 0，导致 g_base ≈ 0，但严重低估了顶级品牌公司的定价权驱动增长。

处理流程：
1. **Step A-B**：Payout < 95% → 标准处理；≥ 95% → 进入特殊处理
2. **Step C**：确定竞争等级（TOP/STRONG/NORMAL），三维度综合判断
3. **Step D**：查表确定 g_floor（TOP-4%/3%, STRONG-2%/1.5%, NORMAL-0%）
4. **Step E**：红旗检测——预收/合同负债连续两季同比下滑时取消 g_floor
5. **Step F**：现金流溢价系数（仅TOP档，OCF/净利润≥1.1→+10%，≥1.0→+5%）
6. **Step G**：g_final = max(g_base, g_floor_final)，与 ROE Cap 取 min

> 竞争等级判定表、g_floor 查表、红旗检测详细流程、现金流溢价规则，见 `references/valuation-special-cases.md`

#### Phase 2.4：周期行业适配增强（⚠ 自动对所有股票执行）

核心逻辑：通过 ROE 标准差自动判断周期性，对强周期公司执行保守处理。

**执行流程**：
1. 计算近 5 年 ROE 标准差（⚠ 必须使用加权ROE，禁止估算）
2. **ROE_std ≥ 6pp** → 强周期信号：
   - ROE_used = min(均值ROE, 最新年度ROE, 行业中位数ROE)，最低为 0
   - Phase 2.5 ④ROE稳定性评分强制 1 分
3. **周期位置折扣**：极度过热→×0.5，偏高估→×0.7，正常→×1.0
4. **动态目标收益率**：ROE_std <3pp→8%，3-6pp→9-10%，≥6pp→10-12%
5. **红旗检测**：周期性重资产行业+资产负债率>70% → 红旗触发

> ROE取值规则、周期位置折扣阶梯、红旗检测详细规则，见 `references/valuation-special-cases.md`

#### Phase 2.5：g_final 增速可信度评估 + 模型适配度评分（⚠ 必须执行）

算出 g_final 后，**必须评估该增速在未来 10 年的实现概率**。

**核心原则：g 取值必须参考历史增速，但绝不能只看 5 年均值**——本质是行业周期判断。

**6 项评估维度（每项 1-3 分）**：

| 维度 | 评估内容 |
|------|---------|
| ① 历史增速 | 多周期 EPS CAGR + 周期阶段判断 → g调整 |
| ② 再投资效率 | 留存利润再投资是否真能创造价值 |
| ③ 行业空间 | 渗透率、增速趋势、天花板 |
| ④ ROE 稳定性 | 近5年 ROE 波动幅度（禁止估算！） |
| ⑤ 模型适配性 | 股票特征与模型假设的匹配程度 |
| ⑥ 利润质量 | EPS 与经营现金流匹配度，是否存在洗澡嫌疑 |

**总分与可信度等级**：15-18→🟢高可信，10-14→🟡中可信，6-9→🟠低可信，0-5→⚫模型不适用

**⑦ 价值实现路径识别**（Phase 2.5 完成后执行）：
- EPS→DPS增长（★★★完美适配）
- PB修复+股息（★★部分适配）
- 回购增厚（★★部分适配）
- 成长股不分红（★不适配）

> 评分标准详解、周期g调整规则、价值路径识别、银行股/高回购公司补充规则，见 `references/credibility-assessment.md`

#### Phase 2.6：估值风险维度（⚠ Phase 2.5 完成后执行）

识别两类"模型无法感知"的风险：

**A. PE 历史分位数检测**：差异化阈值矩阵（ROE稳定→90%分位触发，ROE波动→75%，强周期→60%）

**B. 行业结构性断裂识别**：5类外生断裂信号（监管政策改变、技术替代、需求萎缩、贸易壁垒、盈利模式改变），任一触发则 g=0

> PE分位数计算方式、差异化阈值矩阵、断裂信号处理规则，见 `references/credibility-assessment.md`

#### Phase 2.7：银行股专属修正引擎（⚠ 仅银行股执行）

三项专属修正：①坏账拨备还原 ②PB锚点+估值修复溢价 ③动态Target Yield

最终安全买入价 = max(DDM(含溢价), 股息率锚定价)

> 银行股修正详细规则，见 `references/valuation-special-cases.md`

### Phase 3：筛选条件检验（一票否决）

| 筛选条件 | 要求 | 结果 |
|----------|------|------|
| 历史分红稳定 | 近 5 年每年分红率均 > 30% | ✅ / ⚠️ / ❌ |
| 现金流健康 | 近 3 年平均经营现金流 ≥ 净利润 | ✅ / ⚠️ / ❌ |
| 负债率可控 | 资产负债率平稳或下降；周期性重资产行业须额外满足 ≤70% | ✅ / ⚠️ / ❌ |
| 业务确定性 | 商业模式简单，10年后仍被需要 | 供参考 |

**一票否决**：当"历史分红稳定"或"现金流健康"出现 ❌ 时，安全买入价结论中注明"⚠️ 有一票否决项，建议回避"。

**负债率可控详细判断**：
- 非周期：≤60%→✅，60-70%→⚠️，>70%→❌
- 周期性重资产行业（ROE_std ≥ 6pp）：>70%→⚠️严审（Phase 2.4红旗触发）

## 核心公式速查

```
安全买入价 = [ EPS_curr × (1 + g_adjusted)^10 × Payout ] / target_yield_adjusted

g_base   = ROE_used × (1 - Payout_5y)
# ROE_used = 均值ROE（标准）或 min(近5年ROE)（ROE_std ≥ 6pp时）

g_final 根据 ROE 档位截断：
  ROE > 25%           → min(g_base, 8%)
  15% ≤ ROE ≤ 25%     → min(g_base, 5%)
  ROE < 15%           → min(g_base, 3%)

动态目标收益率（Phase 2.4）：
  ROE_std < 3pp       → target_yield = 8%
  ROE_std 3-6pp       → target_yield = 9-10%
  ROE_std ≥ 6pp       → target_yield = 10-12%

g_adjusted = g_final × 周期折扣系数（Phase 2.4 Step 3）：
  极度过热            → × 0.5
  偏高估              → × 0.7
  正常                → × 1.0
```

## 注意事项

- 目标收益率默认 8%，极其确定的资产（如长电）可调整为 6%-7%
- 模型仅供参考，不构成投资建议
- 数据质量决定结果质量，分红数据缺失时需明确说明
- **银行股专属修正引擎（Phase 2.7）**：三项修正联动，最终安全买入价 = max(DDM(含溢价), 股息率锚定价)
- **g取值必须结合周期判断**：周期高点→g下调，调整末期→g可上修，调整的是g_base截断前的原始值
- **顶级消费品牌享有结构性估值溢价**：ROE持续>25%、品牌护城河极深的公司，市场长期给予PE 25-35倍估值溢价是合理的
- **财务洗澡后必须使用还原EPS**：偏差率>30%→强制双版本输出，取较低者
- **强周期行业强烈不建议使用此模型**：仅作价值下限粗略参考
- **成长股强烈不建议使用此模型**：必须配合PEG判断，PEG<1是被低估信号
- **高回购公司需用总资本回报率辅助判断**
- **PE极端高位时模型可能误导**：Phase 2.6的PE分位数检测是第一道防线
- **行业结构性断裂时历史ROE完全失效**：g直接清零，安全价仅供参考下限
- **财务洗澡检测已自动化**：偏差率>30%自动双版本输出
- **周期性重资产行业需强化三项保护**：保守ROE锚+10-12%目标收益率+资产负债率>70%红旗

> 历史回测案例和教训，见 `references/backtesting.md`
