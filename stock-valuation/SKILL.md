# 内生增长分红估值模型（V1.1）

## 模型简介

本 skill 实现"内生增长分红估值模型"，核心逻辑：
- 以公司的 **ROE**（盈利效率）与 **分红率（Payout）** 推导内生增长率
- 以"十年后分红收益率（Yield on Cost）达到 **8%**"为目标
- 反推**当前安全买入价**

> 参考文档：`references/model_spec.md`

#### 1.2 通用规则（⚠ 关键，严格执行）

**A. 年报时效判断**

- A 股年报截止 **4/30**，查询前先确认当前日期
- 如果当年年报已发布（API 返回截至该年度 12/31 的年报数据），**必须使用最新年报数据**，不要使用 TTM 或旧年报
- 如果年报尚未发布：用最新可用年报数据估值，同时查询最新季度数据写入"业绩说明"章节

**B. 股票代码格式**

- `ts_code` **必须带交易所后缀**：上交所 `.SH`（如 `600519.SH`）、深交所 `.SZ`（如 `000651.SZ`、`002304.SZ`）

**C. 数据年份识别**

- API 返回的多期数据可能混合不同年度，**务必检查每条记录的年份标注**，提取对应年度的数据
- 常见年份字段名：`end_date`、`m_anntime`、报告期

**D. 数据解读防错**

- **"预告" ≠ "实际"**：业绩预告区间 ≠ 年报终值，不能直接用于估值计算
- **避免过度推断**：拿到原始数据后先理解上下文和字段含义，再得出结论
- **数据不完整时降级处理**：如 5 年数据不全可用近 3 年替代，但需在报告中注明

**D2. ⛔ 强制规则：禁止估算，必须实际查询（覆盖全部 11 项数据）**

> **背景**：东阿阿胶（2026-05-07）估值时，只查了最新ROE就估算了近5年ROE标准差，导致④ ROE稳定性评分为"估计3-6pp"而非实际值。同理，任何其他数据项用估算代替查询，都会导致估值失真。
>
> **规则（适用于 Phase 1 全部 11 项数据）**：
>
> 1. **每一项数据都必须通过 API 实际查询**，禁止用最新值估算历史值、禁止拍脑袋估计、禁止用"合理假设"代替实际数据
> 2. **11 项数据清单逐项核对**：①EPS ②股价 ③近5年ROE ④近5年分红 ⑤分红政策 ⑥经营现金流 ⑦资产负债率 ⑧近5年EPS ⑨行业趋势 ⑩最新季报 ⑪年度分红方案（B④按需）——每项都必须在报告中有明确的数据来源标注
> 3. **数据缺失时的降级处理**：若某项数据 API 确实无返回，在报告中注明"{数据项}：API未返回，使用估算值{X}，评分降级"，且对应评分维度最高只能给到该维度满分的 2/3
> 4. **违反此规则属于执行错误**：必须在本次会话的 skill 反思中纠正，并更新 SKILL.md 防止再犯

**E. ⛔ 编码乱码处理（强制规则，不得跳过）**

> **背景**：NeoData API 返回的中文可能因终端编码问题显示为乱码（如 `������������`）。
> **2026-05-07 修复**：`query.py` 已升级为终端输出使用 `ensure_ascii=True`（中文转义为 `\uXXXX`），不再产生乱码。
> **推荐做法**：查询时始终使用 `--output-file` 参数保存结果到文件，再用 Read 工具读取——文件中的 UTF-8 中文完全正确。

```
Step E1：执行查询（推荐方式）
        # 始终保存到文件，避免任何编码问题
        python3 scripts/query.py --query "xxx" --output-file /tmp/result.json --pretty
        
Step E2：读取结果
        使用 Read 工具读取 /tmp/result.json
        → 中文完全正确，无乱码风险

Step E3：备用方案（直接终端输出）
        如未使用 --output-file，终端输出为 \uXXXX 转义格式
        → 数字和英文关键字段可直接提取
        → Unicode 转义序列是可解析的（如 \u4e1c\u963f = "东阿"）
        → 仍可使用 "10\u6d3e" 匹配 "10派"
```

#### 1.3 各数据项详细说明

##### ① 最新年报 EPS

- **取值规则**：必须使用最新年报的**基本每股收益**（非稀释每股收益）
- **优先级**：年报 EPS > 业绩预告 EPS > TTM EPS（降级使用需注明）
- **解读要点**：注意确认 EPS 对应的年度（年报 EPS 对应的是上一会计年度的利润）
- **验证方式**：可交叉验证 `归母净利润 ÷ 总股本 ≈ EPS`

##### ② 最新股价 + 行情指标

- **取值**：最新交易日的收盘价
- **配套查询**：同时获取 PE(TTM)、PB、股息率、年初至今涨跌幅等，写入报告参数表
- **用途**：与安全买入价对比计算安全边际
- **注意**：如果估值时为非交易时段（周末/节假日），使用最近一个交易日的收盘价

##### ③ 近 5 年加权 ROE

- **取值**：**加权平均净资产收益率**（非全面摊薄 ROE）
- **计算方式**：取近 5 个年度的加权 ROE，计算算术平均值
- **注意**：如果最新年报 ROE 异常低（如因一次性亏损），可考虑是否排除，但必须在报告中说明理由
- **API 字段**：`fina_indicator` 中的 `roe` 字段

##### ④ 近 5 年分红数据

- **必须获取的信息**：每年全年每股股利（元）、是否含中期分红、全年分红率
- **计算全年分红率**：`全年每股股利 ÷ 当年 EPS × 100%`

---

**⛔ 强制规则：每次估值必须先确认"是否双次分红"，不得跳过**

A 股分红分为**中期分红**（H1，约 7-9 月公告）和**年度分红**（次年 4-6 月公告），两者独立。API 返回的分红记录**按公告日期排列**，不按归属年度汇总——这是最常见的数据陷阱。

**Step 4-A：先查询全年分红概览（必做）**

使用通用关键词获取近 5 年分红记录：
```
"{公司名}{代码} 2020-2025历年分红送配每股派息分红率年度分红中期分红"
```

**Step 4-B：扫描是否双次分红（必做，不得跳过）**

从 Step 4-A 的返回结果中检查：同一年度是否出现两条分红记录（一条中报 + 一条年报）。

| 判断 | 后续动作 |
|------|---------|
| 同一年度有两条分红记录 | ✅ 确认双次分红 → 转到 Step 4-C |
| 同一年度只有一条，且为"年度分红"/"年报分红" | 可能遗漏中报 → 转到 Step 4-D 单独验证 |
| 同一年度只有一条，且公告日期在 7-9 月 | 大概率只是中报 → 转到 Step 4-D 补查年报分红 |

**Step 4-C：按归属年度汇总双次分红**

对每个年度，将中报每股派息 + 年报每股派息合并为"全年每股股利"：
```
全年每股股利 = 中报每股派息 + 年报每股派息
全年分红率 = 全年每股股利 ÷ 当年 EPS × 100%
```

**Step 4-D：单独验证（对疑似只有一次的年度）**

如果 Step 4-B 不确定是否双次分红，**必须**分别查询中报和年报：
```
关键词1: "{公司名}{代码} {年份}年中期分红方案每股派息"
关键词2: "{公司名}{代码} {年份}年年报分红方案每股派息"
```
确认确实只有一次后，再按单次分红计算。

**Step 4-E：最终校验（必做）**

计算出的全年分红率必须做一次合理性校验：
- 如果分红率 < 20% 且该股票历史上分红率稳定在 30%+ → ⚠️ 极可能遗漏了中报分红
- 如果全年每股股利明显低于预期 → 重新检查是否有中报分红

##### ⑤ 分红政策/承诺/规划

- **查询要点**：同时搜索**当前有效规划**和**历史规划变更记录**
- **确定优先级（从高到低）**：
  1. **公司最新分红规划**（如"2024-2026年分红率不低于75%"）：优先采用，反映管理层未来意图
  2. **⚠ 必须确认承诺是否仍然有效**：搜索"分红规划 变更/修订/取消/调整"，确认是否有新规划替代旧规划
  3. **绝对不能仅凭旧公告就采信**（如洋河 2026 年取消了"不低于70亿元"的硬承诺）
  4. **近 3-5 年实际分红率均值**：在没有明确承诺时使用
  5. **注意区分"年度分红"和"特别分红/中期分红"**：一年分两次的必须加总
- **报告中的标注**：
  - 如有承诺：说明具体内容、承诺期限、是否已兑现
  - 如承诺已变更：⚠ 标注变更时间和新旧差异（这是重大事项，需在报告头部注明）
  - 如无承诺：说明情况，使用实际均值

##### ⑥ 近 5 年经营现金流净额

- **用途**：Phase 3 筛选条件"现金流健康"检验
- **检验标准**：近 3 年平均经营现金流 ≥ 净利润（比值 ≥ 100%）
- **解读要点**：
  - 单年低于 100% 不一定有问题，但**连续多年低于 100% 是警告信号**
  - **经营现金流为负是重大风险**（如洋河 2025 年 -7.63 亿），说明分红可能面临执行困难
  - 计算"净利润现金含量"= `经营现金流净额 ÷ 归母净利润 × 100%`

##### ⑦ 资产负债率 + 有息负债率

- **用途**：Phase 3 筛选条件"负债率可控"检验
- **必须同时查询两个指标**：
  - **资产负债率** = 总负债 ÷ 总资产（一般标准：< 70%）
  - **有息负债率** = (短期借款 + 一年内到期的非流动负债 + 长期借款 + 应付债券) ÷ 总资产
- **判断"借钱分红"嫌疑**：如果资产负债率高 + 有息负债率高 + 分红率高 → ⚠️
- **辅助判断**：财务费用为负（利息收入 > 支出）说明公司资金充裕，不存在借钱分红

##### ⑧ 近 5 年每年 EPS（用于 g_final 可信度评估）

- **用途**：计算 EPS 复合增速 CAGR，与 g_final 对比
- **计算公式**：`CAGR = (EPS_最新年 / EPS_最早年)^(1/年数) - 1`
- **解读**：
  - CAGR ≥ g_final → 历史增速支撑模型假设（高分）
  - CAGR < g_final 的 50% 或持续下滑 → 增速假设不可靠（低分）
- **注意**：此数据可与 ① 合并查询（同一 API 返回多年 EPS）

##### ⑨ 行业趋势信息（用于 g_final 可信度评估）

- **查询方式**：搜索该行业的最新趋势分析，而非特定公司的数据
- **关键词示例**：`"白酒行业 2025 2026 趋势 产量 增速 消费"`、`"空调行业 存量 竞争 格局"`
- **需了解的信息**：
  - 行业处于增长期 / 存量期 / 衰退期
  - 行业产量/规模的近年增速趋势
  - 是否有重大政策变化影响行业（如禁酒令、能效标准等）
  - 主要竞争对手的动态
- **评分参考**：
  - 行业仍有增长空间 → 高分
  - 行业接近饱和但需求稳定 → 中分
  - 行业衰退或深度调整 → 低分

##### ⑩ 最新季度业绩（年报已发布时跳过）

- **用途**：年报尚未发布时，在"业绩说明"章节展示最新季度/前三季/预告数据
- **获取**：最新一季度的营收、归母净利及同比变化
- **注意**：如果已有最新年报数据，此项不需要查询

##### ⑪ 年度分红方案（年报刚发布时补充查询）

- **场景**：年报刚发布（如 4 月底），分红方案可能尚未公告或尚未入库
- **查询**：`"{公司名}{代码} {年份}年报分红方案每股股利派息"`
- **处理**：如果查不到年度分红方案，在报告中注明"年度分红方案待股东大会确认"，用承诺值或历史均值估算

##### ①② 财务洗澡检测 + 偏差熔断机制（⚠ 必须执行，自动触发）

**目的**：在估值计算前，自动检测EPS是否因会计政策选择（激进确认/一次性洗澡）而失真，触发熔断时强制输出双版本报告。

**第一步：自动化偏差计算（每次估值必须执行，无需人工判断）**

> 此步骤在 Phase 1 数据收集完成后自动执行，在查询财报时同步计算，无需用户触发。

**计算还原EPS（OCF法）：**
```
还原EPS = 近3年平均经营现金流净额 ÷ 当期末总股本
```

**计算偏差率：**
```
偏差率 = |报表EPS - 还原EPS| / 报表EPS × 100%
```

**⚠ 偏差熔断阈值：**

| 偏差率 | 熔断等级 | 触发动作 |
|--------|---------|---------|
| **偏差率 ≤ 15%** | ✅ 不触发 | 正常计算，使用报表EPS |
| **15% < 偏差率 ≤ 30%** | ⚠️ 观察级 | 正常计算，同时在报告中列示还原EPS供参考，注明"⚠️ EPS存在轻微失真风险" |
| **偏差率 > 30%** | 🔴 熔断级 | **强制输出双版本报告**（见下方"熔断输出规则"） |

---

**第二步：红旗信号自动扫描（在第一步偏差计算之外，额外检查）**

**A. 经营现金流匹配度异常（自动扫描近5年）**

计算每年「净利润现金含量 = 经营现金流净额 ÷ 归母净利润」：

| 红旗信号 | 含义 | 附加动作 |
|---------|------|---------|
| 连续2年现金含量 < 0.7 | 利润质量差，EPS可能虚高 | 在偏差熔断基础上附加⚠️标注 |
| 单年现金含量暴降至 < 0.3，次年恢复正常 | 疑似洗澡年 | 自动剔除洗澡年，用「近3年均值」重新计算还原EPS |
| 单年EPS同比下滑 > 50% 但现金流正常 | 典型洗澡特征 | 自动标注，报告中说明"疑似一次性洗澡" |

**B. 会计差错更正/追溯调整公告（主动查询）**

查询关键词：`"{公司名}{代码} 会计差错更正 追溯调整 前期会计差错"`、`"{公司名} 前期会计差错更正公告"`。

| 情形 | 附加动作 |
|------|---------|
| 存在公告但调整幅度 ≤ 30% | 在报告中注明，但不触发熔断 |
| **存在公告且调整幅度 > 30%** | **自动触发🔴熔断**，强制双版本输出 |

**C. 董事长/CFO更换 + 业绩暴雷（主动查询）**

查询关键词：`"{公司名}{代码} 董事长 更换 辞职 被查"`、`"{公司名} CFO 财务总监 更换"`。

| 情形 | 附加动作 |
|------|---------|
| 新董事长上任后首份年报业绩暴雷 | 用上任前3年EPS均值替代洗澡后EPS |
| CFO频繁更换（3年换2次以上） | 在筛选条件检验中标记"⚠️ 财务政策不连续" |
| 董事长被查/留置后发布业绩大幅下调 | **自动触发🔴熔断** |

**第三步：还原EPS计算（三选一，按优先级）**

| 方法 | 公式 | 适用场景 | 优先级 |
|------|------|---------|-------|
| **OCF法** | `还原EPS = 近3年平均经营现金流净额 ÷ 总股本` | 通用，最稳健 | ★★★ |
| **均值法**（剔除异常年） | `还原EPS = 近3年EPS均值（剔除洗澡年）` | 无现金流异常但EPS被操纵时 | ★★ |
| **行业对标法** | `还原EPS = 行业平均EPS × 该公司市场份额调整` | 极端异常，数据完全不可信 | ★ |

**第四步：熔断输出规则（🔴自动执行，强制执行）**

```
if 偏差率 > 30% or (B类触发且调整幅度>30%) or C类触发:
    输出版本A = 报表EPS计算的安全买入价（标注"报表版"）
    输出版本B = 还原EPS计算的安全买入价（标注"保守版"）
    安全买入价结论 = min(版本A, 版本B)
    报告中注明："⚠️ 财务洗澡嫌疑，偏差率{X}%>30%，强制双版本输出，取较低者¥XX作为绝对安全价"
else if 15% < 偏差率 ≤ 30%:
    仅输出版本A
    在报告中并列展示还原EPS供参考，注明"⚠️ EPS存在轻微失真风险（偏差率{X}%），还原EPS仅供参考"
else:
    仅输出版本A，无特殊标注
```

### Phase 2：执行估值计算

#### Phase 2.1：100% 分红率特殊处理（⚠ 关键规则）

> **触发条件**：当分红率接近或等于 100%（Payout ≥ 95%）时，执行此步骤。

**核心问题**：标准公式 `g_base = ROE × (1 - Payout)` 中，`(1 - Payout)` 趋于 0，导致 g_base ≈ 0，内生增长模型失效。但这严重低估了**顶级品牌公司**通过定价权驱动的真实增长能力。

**处理逻辑**：

```
Step A：计算标准 g_base
  g_base = ROE × (1 - Payout)

Step B：识别是否触发特殊处理
  if Payout < 95%:
      g_final = min(g_base, Cap)           # 标准处理
      goto Phase 2.5
  else:
      goto Step C                             # 100% 分红特殊处理

Step C：确定公司竞争等级（定价权评估）

  根据以下三个维度综合判断（满足任意两个即可归入该档）：

  | 维度 | TOP | STRONG | NORMAL |
  |------|-----|--------|--------|
  | 市占率/竞争格局 | 绝对垄断（>70%）或品类=品牌 | 龙头地位稳固（30-70%），有明确护城河 | 竞争激烈，无明显护城河 |
  | 定价权 | 有持续提价能力且历史兑现（如茅台/片仔癀） | 有一定议价能力，提价偶尔兑现 | 无定价权，价格战频发 |
  | ROE 水平 | ROE ≥ 20%，且稳定性强 | ROE 15-20%，或 ≥ 20% 但波动大 | ROE < 15% 或 ROE 波动极大 |
  | 典型案例 | 茅台、片仔癀、东阿阿胶 | 腾讯、格力电器 | 大多数普通消费品 |

Step D：查表确定 g_floor

  | 竞争等级 | g_floor（保底增速） | 必要条件（ROE） | 充分条件（定价权/护城河） |
  |---------|-------------------|---------------|------------------------------|
  | **TOP-4%** | **4%**（极强定价权） | ROE ≥ 25%，且稳定性强 | 有持续提价能力且历史兑现（如茅台/片仔癀）；或品类=绝对品牌垄断 |
  | **TOP-3%** | **3%**（稳健） | ROE ≥ 20%，且稳定性强 | 品类=品牌，市占率>70%；或具备较强定价权 |
  | **STRONG-2%** | **2%**（较强惯性） | ROE 20-25%，或 ≥25% 但波动较大 | 龙头地位稳固（30-70%），提价偶尔兑现；有明确护城河 |
  | **STRONG-1.5%** | **1.5%**（稳健） | ROE 15-20% | 有一定议价能力，行业惯性较强 |
  | **NORMAL** | **0%** | 不满足以上条件 | 竞争激烈，无明显护城河，无定价权 |

  > **g_floor 选取规则**：按从上到下顺序匹配，优先尝试更高档。
  > - ROE 是基础门槛，但**定价权/护城河是升档的充分条件**
  > - 例如：ROE=22% 但有明确提价历史 → 可升入 TOP-4%；ROE=18% 但无护城河 → STRONG-1.5% 或 NORMAL

Step E：红旗检测——预收/合同负债连续两季同比下滑（⚠ 必须执行）

> **适用条件**：当公司被识别为 TOP 或 STRONG 档时执行，NORMAL 档跳过。
>
> **核心逻辑**：预收款项/合同负债是品牌消费品（尤其是白酒）的需求先行指标。
> 若该指标连续两季同比下滑，说明渠道/消费者打款意愿减弱，品牌景气度正在下行——
> 即使当前 ROE 极高，g_floor 补偿也可能失效。

```
Step E1：获取近4季预收/合同负债数据
        # 注意：预收款项（老准则）和合同负债（新准则）是同一科目，合并查询
        # 计算公式：预收/合同负债同比 = (本期 - 去年同期) / |去年同期| × 100%
        # 查询关键词：`"{公司名}{代码} {年份}预收款项合同负债科目余额同比变动季报"`

Step E2：判断是否连续两季同比下滑
        if 近2季同比均 < 0：
            红旗触发 = True
            说明："预收/合同负债连续两季同比下滑（Q1:{X}%，Q2:{X}%），需求端走弱，g_floor补偿取消"
        else：
            红旗触发 = False
            说明："预收/合同负债无连续下滑信号（Q1:{X}%，Q2:{X}%），g_floor补偿维持"

Step E3：确定 g_floor_final
        if 红旗触发：
            g_floor_final = 0
        else：
            g_floor_final = g_floor（沿用Step D结果）
```

**报告标注**（在 Step 1 估值计算过程中必须展示）：
```
| 预收/合同负债趋势 | Q1同比 {+X%/-X%}，Q2同比 {+X%/-X%} | 红旗信号：{触发/未触发} |
| g_floor 最终取值 | {X}%（原始值 {X}%，红旗影响：{取消/维持}） | |
```

Step F：现金流溢价系数（⚠ 仅对 TOP 档公司执行）

> **核心逻辑**：若 OCF/净利润 ≥ 1.1，说明公司利润几乎全部转化为现金，极强议价权叠加优质现金流，
> 安全买入价应给予上浮。若 OCF/净利润在 1.0-1.1 之间，给予温和溢价。
>
> **适用条件**：仅当竞争等级为 TOP 档时执行。STRONG 档及以下不适用。

```
Step F1：计算近3年平均 OCF/净利润
        OCF_净利润比 = 近3年平均经营现金流净额 ÷ 近3年平均归母净利润

Step F2：判断溢价档位
        if OCF_净利润比 ≥ 1.1：
            现金流溢价系数 = 10%
            说明："OCF/净利润={X}≥1.1，极强现金流+议价权，溢价10%"
        elif OCF_净利润比 ≥ 1.0：
            现金流溢价系数 = 5%
            说明："OCF/净利润={X}≥1.0，优质现金流，溢价5%"
        else：
            现金流溢价系数 = 0%
            说明："OCF/净利润={X}<1.0，无溢价"

Step F3：计算含溢价安全买入价（在 Phase 2.1 结束时执行）
        安全买入价（DDM基础） = [ EPS_curr × (1 + g_adjusted)^10 × Payout ] / target_yield_adjusted
        安全买入价（含现金流溢价） = 安全买入价（DDM基础） × (1 + 现金流溢价系数)
        # 注意：此溢价在最终安全买入价中取 max 时一并考虑
```

**报告标注**（在 Step 4 或估值结论中展示）：
```
| OCF/净利润均值（近3年） | X.XX | ≥1.1=溢价10%，≥1.0=溢价5%，<1.0=无溢价 |
| 现金流溢价系数 | +{X}% | {说明} |
| 安全买入价（含现金流溢价） | ¥XX.XX | DDM基础价 ¥XX × (1+{X}%) |
```

Step G：最终 g 取值

  g_final = max(g_base, g_floor_final)

  # 实际效果：
  # - TOP 公司：g_final = max(≈0, 3-4%) = 3-4%（红旗触发则清零）
  # - STRONG 公司：g_final = max(≈0, 1.5-2%) = 1.5-2%
  # - NORMAL 公司：g_final = 0%

  # 注意：最终仍需与 ROE 档位 Cap 比对，取较小值
  # g_final = min(g_final, Cap)
```

**Phase 2.1 完整执行流程（更新后）**：

```
Step A：计算标准 g_base
Step B：Payout ≥ 95%？ → 是：进入 Step C；否：标准处理 → Phase 2.5
Step C：确定竞争等级（TOP/STRONG/NORMAL）
Step D：查表确定 g_floor
Step E：红旗检测（预收/合同负债连续两季下滑）→ 取消/维持 g_floor → g_floor_final
Step F：现金流溢价系数（仅TOP档）→ 计算溢价幅度
Step G：g_final = max(g_base, g_floor_final) → 与 ROE Cap 取 min
        → 安全买入价（含现金流溢价）= DDM结果 × (1 + 现金流溢价系数)
        → 继续 Phase 2.4
```

**报告中的标注要求**：

当 Payout ≥ 95% 时，在估值计算过程（Step 1）中必须说明：
1. 公司竞争等级判定（TOP / STRONG / NORMAL）及判定依据
2. 选用的 g_floor 值及理由
3. 红旗检测结果（预收/合同负债趋势 + 是否取消 g_floor）
4. 现金流溢价系数（仅TOP档）
5. 最终 g_final 取值及是否触发了 ROE 档位截断

**⚠ 常见错误警示**：

| 错误做法 | 后果 | 正确做法 |
|---------|------|---------|
| 对茅台、东阿阿胶等顶级公司直接设 g=0 | 安全买入价严重低估，无法体现品牌价值 | 对 TOP 公司使用 g_floor=3-4% |
| 对没有定价权的普通公司也设 g=3% | 安全买入价过于乐观 | 对 NORMAL 公司维持 g=0% |
| 不区分 TOP/STRONG/NORMAL | 分类模糊，结论主观 | 按三维度综合打分，满足任意两个条件即归类 |

#### Phase 2.4：周期行业适配增强（⚠ 适用所有股票，自动触发）

> **触发条件**：此步骤**自动对所有股票执行**，通过ROE标准差判断是否触发特殊处理。无需用户指定，自动判断。

**核心问题**：标准公式假设 ROE 相对稳定，但对钢铁、煤炭、有色等强周期行业，ROE 可从 +30% 暴跌至负数。用"均值ROE"或"最低ROE"计算安全价，在行业下行期都极具误导性。

##### 第一步：计算近 5 年 ROE 标准差

> ⚠ **ROE 类型规定**：统一使用**加权平均净资产收益率**（`roe`），**不得使用摊薄 ROE（`roe_wa` vs `roe_today` 中的摊薄字段）**。两者差异在 ROE > 20% 时可能超过 2pp，直接影响 ROE_std 判断。

```
ROE_std = std(加权ROE_年1, 加权ROE_年2, ..., 加权ROE_年5)
# 数据来源：近5年年报 `fina_indicator` 表的 `roe` 字段
```

##### 第二步：ROE 异常检测（强制触发）

| 触发条件 | 说明 |
|---------|------|
| **ROE_std ≥ 6 个百分点** | 强周期信号，触发均值回归强制替代 |
| **ROE_std < 6 个百分点** | 无需特殊处理，标准计算 |

**触发后的 ROE 取值规则（⚠ 仅适用于 ROE_std ≥ 6pp 的公司）：**

> **核心原则**：ROE_std ≥ 6pp 说明公司具有强周期性——5年均值可能因行业繁荣顶而虚高，
> 必须取"均值、最近一年、行业中位数"三者最小值作为保守锚。
>
> **边界条件**：`# 仅适用于 ROE_std ≥ 6pp 的周期性公司；ROE_std < 6pp 时，无论何种行业，均用均值 ROE`

```
if ROE_std ≥ 6:
    # 仅适用于周期性行业；ROE_std < 6pp 时，非周期性行业用均值 ROE

    # 步骤A：构建候选 ROE 集合（仅周期性行业执行此步骤）
    candidates = [均值ROE, 最新年度ROE, 行业中位数ROE]

    # 步骤B：取三者最小值（保守锚）
    ROE_used = min(candidates)
    说明："ROE_std={X}pp≥6pp，周期性行业，取5年均值{XX}%、
          最近一年ROE{XX}%、行业中位数{XX}%三者最小值{XX}%作为保守锚"

    # 步骤C：若最小 ROE 为负，用 0 替代（防止 ROE × (1-Payout) 出现荒谬结果）
    if ROE_used < 0:
        ROE_used = 0
        说明："最小ROE为负，以0替代，避免负增长陷阱"

    # 步骤D：数据来源说明（⚠ 统一使用加权ROE，不得用摊薄ROE）
        5年均值ROE：近5年年报**加权ROE**算术平均
        最近一年ROE：最新年度**加权ROE**
        行业中位数ROE：查询关键词 `"{行业名}行业中位数ROE {年份}"`（行业数据若无特殊说明默认也为加权口径）
        注：若无法获取行业中位数，仅用"5年均值"和"最近一年"两者取小值，并在报告中注明
else:
    # 非周期性行业：使用均值ROE，不引入行业中位数比较
    ROE_used = 均值ROE（标准处理）
    无需特殊处理
```

**⚠ ROE_std ≥ 6pp 的公司在 Phase 2.5 的④ ROE稳定性维度自动评1分**

##### 第三步：周期位置折扣系数（当行业处于繁荣顶时）

即使 ROE_std < 6pp，若**周期位置处于历史高位**，仍需对 g_base 执行折扣。

**量化指标获取：**

| 指标 | 查询方式 | 数据来源 |
|------|---------|---------|
| 行业产能利用率 | `"钢铁行业产能利用率 {年份} {季度} 过高 历史高位"` | 国家统计局、工信部、行业协会 |
| 行业库存周期 | `"煤炭行业库存周期 主动补库 被动去库 阶段"` | 我的钢铁网(Mysteel)、中国煤炭市场网 |
| 主要商品价格分位数 | `"{螺纹钢/铝/铜/煤炭} 价格 历史分位数 {年份}"` | Wind、Choice、行业协会 |
| 宏观 PMI 库存分项 | `"中国PMI 库存 {年份} {季度} 上升 下降"` | 国家统计局 |

**阶梯折扣系数：**

| 周期位置信号 | 判断依据 | 折扣系数 |
|------------|---------|---------|
| **极度过热（→ 0.5）** | 产能利用率 > 历史90分位；价格处于历史前10%分位；行业整体利润率创历史新高 | `g_adjusted = g_base × 0.5` |
| **偏高估（→ 0.7）** | 产能利用率在75分位~90分位；价格处于历史60-90%分位；库存周期处于主动补库顶 | `g_adjusted = g_base × 0.7` |
| **正常/偏低（→ 不打折）** | 产能利用率低于75分位；价格处于历史中低位；库存周期处于去库/被动去库阶段 | 不打折，使用原 g_base |

**计算示例：**

```
假设 g_base = 5%，周期偏高估（折扣 0.7）
→ g_adjusted = 5% × 0.7 = 3.5%
→ 代入安全买入价公式
```

**⚠ 注意**：周期位置折扣**叠加在 ROE_std 触发规则之后**：
- ROE_std ≥ 6pp：先用保守锚（min均值/最近一年/行业中位数）计算 g_base，再判断是否需要折扣
- ROE_std < 6pp：用均值ROE计算 g_base，再判断是否需要折扣

##### 第四步：红旗指标——周期性重资产行业高杠杆风险（⚠ 适用于 ROE_std ≥ 6pp 公司）

> **核心逻辑**：在化工、有色、航运等周期性重资产行业，高杠杆是毁灭性风险。
> 即使利润尚可，高负债率 + 行业下行 = 债务危机风险骤增。
> 此类公司须在 Phase 3 筛选条件检验中执行"严审"。

```
Step 4A：判断是否为周期性重资产行业
        if ROE_std ≥ 6pp and 行业属于以下类别：
            化工/有色/航运/钢铁/煤炭/建材/造纸/玻璃
            → 周期性重资产行业标记 = True
        else：
            → 周期性重资产行业标记 = False，跳过此步骤

Step 4B：获取资产负债率
        # 使用 Phase 1 已获取的资产负债率数据
        资产负债率 = 最新年度资产负债率（总负债 ÷ 总资产）

Step 4C：判断红旗
        if 周期性重资产行业标记 == True：
            if 资产负债率 > 70%：
                红旗触发 = True
                说明："周期性重资产行业（{行业名}）+ 资产负债率{X}%>70%，高杠杆风险极大"
                → 在 Phase 3 "负债率可控"行标注"⚠ 严审"，结果降一档
            else：
                红旗触发 = False
                说明："周期性重资产行业，资产负债率{X}%≤70%，高杠杆风险可控"
        else：
            无需执行红旗检测
```

**⚠ 红旗触发对 Phase 3 的影响**：

当 Step 4B 触发红旗时，Phase 3 的"负债率可控"检验结果从默认档位降一档：

| 资产负债率（周期性重资产） | Phase 3 标注 | 结果 |
|-------------------------|-------------|------|
| ≤ 60% | 正常 | ✅（负债率可控）|
| 60-70% | 观察 | ⚠️（负债率偏高，周期下行有压力）|
| **> 70%** | **⚠ 严审（红旗触发）** | **⚠️（高杠杆+高波动，一票否决风险大）** |

> ⚠ **Phase 3 一票否决强化**：当红旗触发时，"负债率可控"列结果降为 ⚠️，且在 Phase 3 结论中增加：
> "⚠️ 周期性重资产行业 + 资产负债率 > 70%，高杠杆在行业下行期存在债务风险，建议审慎评估。"

**Phase 2.4 执行流程总结：**

```
Step 1：计算 ROE_std
Step 2：ROE_std ≥ 6pp？
    → 是：ROE_used = min(均值ROE, 最新年度ROE, 行业中位数ROE) = max(最小ROE, 0)
           Phase 2.5 ROE稳定性=1分；目标收益率调高至10-12%
    → 否：ROE_used = 均值ROE；继续
Step 3：判断行业周期位置
    → 极度过热：g_adjusted = g_base × 0.5
    → 偏高估：g_adjusted = g_base × 0.7
    → 正常：不打折，g_adjusted = g_base
Step 4：红旗检测（仅 ROE_std ≥ 6pp 的周期性公司执行）
    → 资产负债率 > 70% → 红旗触发 → Phase 3 严审
Step 5：g_adjusted 与 ROE 档位 Cap 取 min → g_final
Step 6：确定动态目标收益率（见下方）
Step 7：代入安全买入价公式（使用 target_yield_adjusted）
Step 8：继续 Phase 2.5 → Phase 2.6 → Phase 2.7 → Phase 3
```

**动态目标收益率（target_yield_adjusted）说明：**

| ROE_std 分档 | 目标收益率 | 逻辑 |
|-------------|----------|------|
| < 3pp（ROE稳定） | **8%（不变）** | 模型输入可靠，无需额外补偿 |
| 3-6pp（ROE波动） | **9-10%** | 轻度风险补偿，提高安全边际要求 |
| ≥ 6pp（强周期） | **10-12%** | 重度风险补偿；与Phase 2.4的最低ROE锚共同作用，安全价极度保守 |

> **贵州茅台2018年11月误判的教训**：PE 28倍看似"历史中位"，但对ROE 30%+的顶级消费品牌，实际是极好的买点。动态目标收益率可以缓解此类问题——对ROE极稳定的公司维持8%目标，对ROE波动公司提高要求，本身就是对"历史不代表未来"的系统性补偿。

**⚠ 注意**：Phase 2.6（估值风险维度）基于 Step 5 确定的 target_yield_adjusted 计算调整后安全价，两者协同作用。

##### Phase 2.4 执行后的报告标注要求

触发 Phase 2.4 特殊处理时，必须在"二、估值计算过程"的 Step 1 中说明：
1. 近5年 ROE 标准差及是否触发均值回归强制替代
2. 若使用了最低ROE（而非均值），说明理由："近5年ROE标准差{X}% > 6pp，均值ROE{X}%无法代表公司常态盈利能力，使用保守锚最低ROE {X}%"
3. 若使用了行业中位数，说明数据来源
4. 若触发了周期位置折扣，说明具体指标值和折扣系数

#### Phase 2.5：g_final 增速可信度评估 + 模型适配度评分（⚠ 必须执行）

算出 g_final 后，**必须立即评估该增速在未来 10 年的实现概率**。g_final 是模型的核心假设——如果这个增速根本无法实现，安全买入价就是空中楼阁。

同时，Phase 2.5 的六维评分汇总后，将在"三、估值结论"顶部以**🎯 模型适配度徽章**形式呈现，一眼可见该安全买入价对该股票的参考权重。

**⚠ 核心原则：g 取值必须参考历史增速，但绝不能只看 5 年均值**

> **根本逻辑**：用"近5年EPS CAGR"前瞻"未来10年增速"，本质上是**行业周期判断**：
> - 周期高点 → 往后增速必然下行 → g要往下调
> - 周期低点/调整末期 → 往后增速必然上行 → g可以往上调
> - 增速平稳 → g接近历史均值
>
> **常见错误**：仅用"近5年EPS CAGR"与g_final对比，得出"历史增速支撑/不支持"的结论。这是严重滞后的判断——5年均值既不反映当前位置（周期顶/底），也不反映均值回归方向。
>
> **正确做法**：
> 1. 分析多周期增速（1年/2年/3年/5年），判断**趋势方向**
> 2. 结合**行业周期判断**，确定当前处于**周期哪个阶段**
> 3. 根据周期位置，**对g_base/g_final做调整**（向上或向下），而非仅做可信度评分

**评估维度（6 项，每项 1-3 分）：**

| 维度 | 评估内容 | 3 分（高可信） | 2 分（中可信） | 1 分（低可信） |
|------|---------|--------------|--------------|--------------|
| **① 历史增速** | 多周期 EPS CAGR + 周期阶段判断 → g调整 | ①+②综合：5年均CAGR≥g_final，**且**周期处于平稳或上升通道，无均值回归风险；或：周期处于调整末期，g可上修 | ①+②综合：5年均CAGR接近g_final，周期方向不明确；或：均值回归风险中等 | ①+②综合：**周期处于历史高位**，近期增速显著高于长期中枢，均值回归压力极大；或：**EPS近3年持续下滑**，周期底部未现 |
| **② 再投资效率** | 留存利润再投资是否真的能创造价值 | 高分红率（≥70%）：不依赖再投资，g_final 天然容易实现 | 中分红率（40%-70%）：适度依赖再投资 | 低分红率（<40%）：高度依赖再投资，需优质投资机会 |
| **③ 行业空间** | 行业渗透率、增速趋势、天花板 | 行业仍有增长空间或稳健需求 | 行业接近饱和但需求稳定 | 行业衰退或深度调整、需求萎缩 |
| **④ ROE 稳定性** | 近 5 年 ROE 波动幅度 | ROE 标准差 < 3个百分点 | ROE 标准差 3-6个百分点 | ROE 标准差 > 6个百分点或单年骤降 > 10个百分点 |
| **⑤ 模型适配性** | 股票特征与模型假设的匹配程度（见下方详细说明） | 适配：高分红+稳定/低增长，模型核心逻辑完全适用 | 部分适配：需要调整输入或解释口径（如含回购的综合分红率、跨币种折算） | 不适配：模型安全价仅作参考下限，不能作为决策依据 |
| **⑥ 利润质量** | EPS与经营现金流匹配度，是否存在洗澡嫌疑 | 近3年OCF/净利润均值 ≥ 0.9，无洗澡嫌疑 | 0.7-0.9，或存在轻微异常（如单年现金含量<0.7但次年恢复） | < 0.7 或存在明显洗澡嫌疑（如董事长更换后业绩暴雷+会计差错更正） |
> ⚠️ **④ ROE 稳定性评分强制规则**（对应 Phase 1 通用规则 D2）：
> - 必须使用 Phase 1 实际查询的**5个年度加权ROE**计算标准差，**禁止估算**
> - 若某年ROE数据缺失，在报告中注明"{X}年ROE缺失，使用{N-1}年数据计算"，且评分降级（稳定性维度最高2分）
> - 违反此规则属于执行错误，直接导致模型适配度总分失真

**总分与可信度等级：**

> **关于 g_final 调整的特殊规则（⚠ 核心补充）**：
>
> 在执行 6 项评分前，**必须先判断周期位置并决定是否对 g_base 做调整**：
>
> | 当前所处周期阶段 | 典型信号 | g 调整方向 | 调整幅度 |
> |-----------------|---------|----------|---------|
> | **周期高点/繁荣顶** | 5年均CAGR >> 长期中枢；近期增速高于均值；行业产能/库存周期高位 | **g往下调**（均值回归压力） | 下调 1-3 个百分点 |
> | **周期调整中/下行期** | EPS/营收近2-3年持续下滑；行业需求疲软；产能利用率下降 | **g维持或小幅下调**（下行趋势未止） | 不调整或下调 1-2 个百分点 |
> | **周期调整末期/底部** | EPS已深度下滑（-20%以上）；行业政策底出现；竞争对手退出；需求指标触底 | **g可以上修**（均值回归向上弹性大） | 上调 1-3 个百分点 |
> | **周期平稳/中位** | 各周期增速趋于收敛；行业需求平稳；增速波动小 | **g接近历史均值** | 不调整 |
>
> **调整原则**：
> - 调整的是 **g_base 截断前的原始值**，而非截断后的 g_final
> - 调整后如果仍超过ROE档位上限，继续截断
> - 周期判断是主观的，应在报告中明确说明判断依据和调整理由
> - 若周期位置不明确（最常见），**默认不调整**，保留模型原始输出

| 总分 | 等级 | 含义 | 报告标记 |
|------|------|------|---------|
| 15-18 | 🟢 **高可信** | g_final 实现概率大，模型结果可靠 | 🟢 高可信 |
| 10-14 | 🟡 **中可信** | g_final 有一定不确定性，需关注风险因素 | 🟡 中可信 |
| 6-9 | 🟠 **低可信** | g_final 实现困难，安全买入价可能偏高 | 🟠 低可信 |
| 0-5 | ⚫ **模型不适用** | 该股票类型不适合此模型，安全价仅供参考下限 | ⚫ 不适用 |

**⑦ 价值实现路径识别（⚠ 核心补充，必须在 Phase 2.5 完成后执行）**

> 在完成 6 项评分后，**立即识别该股票的主要回报来源**，决定模型对这只股票的可靠程度。用户需要先知道"这只股票值不值这个模型"，再去看具体数字。

**四类价值实现路径：**

| 回报来源 | 典型股票 | 模型适配性 | 核心判断逻辑 |
|---------|---------|----------|------------|
| **EPS增长 → 分红增长** | 格力电器、高分红白马、部分消费品牌 | ★★★ 完美适配 | EPS持续增长 → DPS持续增长 → 模型核心逻辑完全适用 |
| **PB估值修复 + 股息率** | 银行、保险、极低估周期股 | ★★ 部分适配（需补充PB框架） | 股价<净资产时买入 → PB修复驱动回报；低增速时股息率成为主要回报 |
| **回购注销 → EPS增厚** | 腾讯控股、优质科技股 | ★★ 部分适配（需用综合资本回报） | 持续回购注销 → 股本减少 → EPS增厚；分红率低但资本配置高效 |
| **EPS增长（不分红）** | 成长股、医药CXO、科技 | ★ 不适配（改用PEG或DCF） | 不分红则无DPS增长，分红模型完全失效 |

**识别方法（三问法）：**

```
Q1：该股票是否持续现金分红（近3年每年分红）？
    → 否 → 成长股路径（★不适配，跳出模型）

Q2：该股票是否持续大额回购注销（近3年回购金额≥净利润30%）？
    → 是 → 高回购路径（★★部分适配，用总资本回报率补充）

Q3：该股票是否为银行/保险，或PB处于历史低位？
    → 是 → 银行股路径（★★部分适配，用PB分位数+股息率补充）

Q1+2+3 均为"否" → 标准路径（EPS→DPS增长，★★★完美适配）
```

**路径识别后的处理规则：**

| 路径 | 在报告中标注 | 模型安全价定位 |
|------|-----------|-------------|
| **EPS→DPS增长** | "价值实现路径：EPS内生增长 → 分红增长，模型适配" | 模型结果可靠 |
| **PB修复+股息** | "价值实现路径：PB估值修复+股息率，模型结果作参考下限" | 仅作下限，配合PB分位数和股息率分析 |
| **回购增厚** | "价值实现路径：回购注销驱动EPS增厚，模型结果作参考下限" | 仅作下限，需用总资本回报率交叉验证 |
| **成长股（不分红）** | "价值实现路径：EPS增长但无分红，模型不适用" | 仅供参考下限，建议改用PEG或DCF |

**⑦ 评分规则补充：**

| 路径 | 适配性评分影响 |
|------|--------------|
| EPS→DPS增长 | 按现有表格正常评分（★★★ → 3分，★★ → 2分） |
| PB修复+股息 | 在现有评分基础上，补充PB分位数+股息率框架（见下方银行股补充分析） |
| 回购增厚 | 在现有评分基础上，补充总资本回报率（见下方高回购公司补充规则） |
| 成长股 | 维持1分或0分，标注"建议改用PEG < 1为低估信号" |

**⑤ 模型适配性——详细评分指南：**

本模型以 **DPS（每股现金分红）** 为核心输入，推导逻辑为 `ROE × 留存率 → 内生增长 → DPS 增长 → 安全价`。因此模型天然适配**高分红、低增长、业务稳定的成熟价值股**。对于其他类型股票，需评估适配程度：

**股票类型与模型适配性：**

| 股票类型 | 典型特征 | 适配度 | 评分建议 | 处理方式 |
|----------|---------|--------|---------|---------|
| **TOP 品牌股（100%分红）** | ROE ≥ 20%、绝对垄断/品类=品牌、定价权历史兑现（如茅台、片仔癀、东阿阿胶） | ★★★ 完美适配（经特殊处理） | 3 分 | Phase 2.1 给出 g_floor=3-4%，品牌定价权对冲零留存；模型结果可靠 |
| **高分红价值股** | 分红率 ≥50%，ROE 稳定，增长温和（如格力、分众、高速公路） | ★★★ 完美适配 | 3 分 | 直接使用模型结果 |
| **稳定白马股** | 分红率 30-50%，ROE 中高，增长中等（如部分银行） | ★★★ 基本适配 | 2-3 分 | 正常使用，注意增长假设 |
| **高回购低分红股** | 现金分红率低，但大量回购注销（如腾讯控股） | ★★ 部分适配 | 2 分 | **必须使用"总资本回报率"替代纯现金分红率**。查询近3年回购金额：`"{公司名}{代码} {年份}股份回购 注销 金额"`。计算：`总资本回报率 = (现金分红 + 回购注销金额) ÷ 净利润`。同时将回购注销视为等效增速：`g_repurchase = (近3年平均回购金额 ÷ 当期末总股本) ÷ 当前EPS`，作为 g_base 的辅助参考（不改变 g_final 正式值）。跨币种公司需统一币种后计算。报告结论中注明："⚠ 存在大额回购，安全买入价仅供参考下限；总资本回报率{X}% 供参考"。 |
| **成长股** | 分红率 <20%，利润高增长（如科技、医药） | ★ 不适配 | 1 分 | 模型安全价仅作参考下限，g_final 严重低估实际潜力。**必须补充PEG维度**：查询当前PE(TTM)和预期EPS增速：`"{公司名}{代码} 分析师一致预期 净利润增速 预测 2025 2026"`。判断：PEG < 1 → 成长性被低估，模型安全价偏低但成长价值存在；PEG 1-2 → 成长预期合理偏高；PEG > 2 → 确认高估。在报告中注明："⚠ 模型对成长股不适用，安全价仅供参考下限；建议同时参考PEG判断"。 |
| **强周期行业**（钢铁/煤炭/有色/化工） | 利润随行业周期剧烈波动，ROE可从+30%到负值，EPS极度不稳定 | ☆ 模型不适用 | **0 分** | **强烈不建议使用此模型**。无论输入如何调整，安全买入价的计算结果都不具备参考价值。正确做法：① 使用 PB-ROE 框架（净资产为锚，规避EPS波动）；② 使用 DCF 自由现金流折现（以产能周期而非会计利润为基础）；③ 如坚持用分红模型，以"还原EPS（OCF/总股本）"作为保守锚，周期位置折扣系数强制设为0.5。Phase 2.4 的 ROE_std 预警在此类公司应自动触发。 |
| **周期股** | 利润波动大，分红不稳定（如钢铁、煤炭） | ★ 不适配 | 1 分 | ROE/EPS 周期波动使模型失效，不建议使用 |
| **金融/银行股** | ROE 高但受监管约束，分红率稳定但增速受限 | ★★ 部分适配 | 2 分 | ⚠ **回测验证（招商银行 2023-11）**：安全买入价低估判断正确（2.5年涨幅66%），但实际上涨驱动力是**PB估值修复 + 股息率**，而非EPS增长（近乎零增长）。模型会系统性低估银行股。**必须在Phase 2.5结束后执行银行股补充分析框架**（见本节末尾专门说明）：① PB历史分位数：当前PB vs 近5年PB区间；② 股息率安全垫：当前股息率是否 ≥ 目标收益率（8%）；③ 综合判断：两个条件满足其一 → 模型可信度从🟡升🟢，并注明"本轮回报主要由PB修复+股息驱动，模型安全价作为价值锚有效"。 |

**港股/跨币种特殊处理：**

- 如果 EPS（利润表）和分红/股价的计价币种不同（如腾讯：EPS 人民币，分红/股价港币），**必须统一币种后计算分红率**
- 使用最新汇率折算，并在报告中注明汇率和折算方式
- 币种差异增加一层不确定性，评分时从 3 分降为 2 分

**评分规则总结：**
- 分红方式为**纯现金**且高分红 → 3 分
- 分红方式含**回购**或需要**跨币种折算** → 2 分
- 模型核心假设与股票特征**根本不匹配**（成长股/周期股/低分红）→ 1 分

**⑥ 利润质量——详细评分指南：**

利润质量维度用于识别EPS是否因会计政策选择（激进确认/一次性洗澡）而失真。

**评分标准详解：**

| 分值 | 情形 | 典型案例 |
|------|------|---------|
| **3 分** | 近3年OCF/净利润均值 ≥ 0.9，无洗澡嫌疑 | 茅台、格力等现金流强劲且稳定的公司 |
| **2 分** | 0.7-0.9，或存在轻微异常（如单年现金含量<0.7但次年恢复） | 行业调整年，现金流暂时偏弱但趋势向好 |
| **1 分** | < 0.7 或存在明显洗澡嫌疑（董事长更换后业绩暴雷+会计差错更正公告） | 五粮液2025年（曾从钦被查后大幅调减前期收入） |

**红旗信号（必须主动查询）：**
1. **会计差错更正公告**：搜索"{公司名} 会计差错更正"，如发现且调整幅度>30%，不得评3分
2. **董事长/CFO异常更换**：董事长被查/留置、CFO频繁更换（3年换2次以上），需降级评分
3. **单年EPS暴降>50%但现金流正常**：典型的"一次性洗澡"信号，需使用还原EPS

**洗澡后处理**：若评分≤2分，必须在报告中使用"还原EPS"重新计算安全买入价，并在估值结论中注明⚠️利润质量警示

#### Phase 2.6：估值风险维度——PE分位数 + 行业结构性断裂识别（⚠ 必须在 Phase 2.5 完成后执行）

> **目的**：识别两类"模型无法感知"的风险：①估值收缩风险（PE处于历史极端高位）；②行业逻辑根本性重构（历史数据完全失效）。

##### A. PE历史分位数检测

> 贵州茅台2018年11月案例教训：PE 28倍看似"历史中位"，但对ROE 30%+的顶级品牌，实际是极好买点。PE分位数需要结合ROE稳定性做差异化阈值。

**数据获取：**

查询近5年PE(TTM)历史数据：
```bash
python3 $QUERY_SCRIPT --query "{公司名}{代码} 2021-2025历年PE市盈率TTM滚动 每月历史" --data-type api
```

同时查询当前PE：
```bash
python3 $QUERY_SCRIPT --query "{公司名}{代码} 最新PE市盈率TTM滚动 当前" --data-type api
```

**分位数计算（手动）：**

将API返回的近5年月度PE数据按升序排列，计算当前PE在历史数据中的分位数：
```
PE分位数 = 当前PE在历史序列中的排名 ÷ 总数据点数 × 100%
```

**差异化阈值矩阵：**

| ROE_std | PE触发阈值 | 含义 | 报告标记 |
|---------|----------|------|---------|
| < 3pp（ROE稳定） | PE > 历史90%分位 | 估值溢价极端，历史极端偏高 | ⚠️ PE处于历史前10%，估值收缩风险极大 |
| 3-6pp（ROE波动） | PE > 历史75%分位 | 估值偏高，应提高警惕 | ⚠️ PE处于历史前25%，存在估值收缩风险 |
| ≥ 6pp（强周期） | PE > 历史60%分位 | 周期顶部+高PE，双重风险 | ⚠️ PE处于历史前40%，周期与估值双重风险叠加 |

**PE分位数 × ROE稳定性综合判断：**

| 情形 | 含义 | 行动 |
|------|------|------|
| ROE稳定 + PE高位 | 优质公司但估值极端，安全价仍然有效，但须耐心等待 | 报告标注⚠️，不调整g或target_yield |
| ROE波动 + PE高位 | 高估+基本面不稳，最危险组合 | 安全价需要折扣（target_yield已自动上浮），报告加⚠️ |
| ROE极不稳定 + PE高位 | 强周期行业+周期顶+高PE，直接输出"⚠ 强烈不建议买入" | 安全价仅供参考下限，需等待周期见底 |

##### B. 行业结构性断裂识别（历史ROE可能完全失效）

> **触发条件**：即使ROE_std看起来正常，若出现以下任一信号，历史ROE可能完全失去参考价值，模型必须进入"极度保守模式"。

**外生断裂信号清单（必须主动查询）：**

| 信号类型 | 查询关键词 | 后果 |
|---------|----------|------|
| **行业监管政策根本性改变** | `"{行业名} 监管政策 重大调整 禁止 限制"` | 竞争格局重构，盈利能力逻辑改变 |
| **重大技术替代威胁** | `"{行业名} 新技术替代 颠覆 产品替代 技术路线变化"` | 现有产品面临淘汰风险 |
| **需求侧结构性萎缩** | `"{行业名} 需求下降 人口老龄化 渗透率见顶 市场规模萎缩"` | 行业长期增速趋势性下行 |
| **出口依赖高且贸易壁垒升级** | `"{行业名} 出口关税 加征 贸易战 壁垒 2025 2026"` | 盈利不确定性大幅提升 |
| **行业盈利模式根本改变** | `"{行业名} 商业模式变化 盈利模式转变"` | 原有ROE水平不可持续 |

**处理规则：**

```
if 任一断裂信号存在：
    g_used = 0（或极低值，0-1%）
    报告标题加⚠️："⚠ 行业结构性断裂风险，当前历史数据可能已失效"
    安全买入价 = (EPS × Payout) / target_yield_adjusted
    报告中注明："⚠ 行业逻辑重构，g假设为0（不做内生增长），安全价仅供参考下限"
else:
    正常执行，不调整
```

**⚠ 注意**：行业断裂识别需要主观判断，模型只能给出"检测信号+建议"，最终判断需在报告中说明。用户若发现行业存在重大结构性变化，应主动说明，模型据此调整。

> **核心逻辑**：银行股的特质在于受监管的资本充足率约束，利润增长往往不如分红和估值修复重要。模型需针对银行的"类债属性"和"藏利润行为"进行专属修正。

##### 第一步：坏账拨备还原（识别"藏利润"行为）

**判断逻辑**：

```
Step A：获取银行拨备覆盖率 + 行业均值
        行业均值参考：五大行均值约 200-250%；全国股份制均值约 180-220%

Step B：判断是否"超额拨备"（藏利润）
        if 拨备覆盖率 > 行业均值 × 1.3（约为 250-300%+）：
            说明：拨备覆盖率远超行业均值，存在主动"藏利润"行为
            还原比例 = min(50%, (拨备覆盖率 - 行业均值) / 拨备覆盖率 × 100%)
            还原金额 = 当年计提拨备 × 还原比例
        else：
            无需还原

Step C：计算还原 EPS
        还原EPS_Bank = 报表EPS + (还原金额 × 75% ÷ 总股本)
        # 75% 为边际税率（还原税后利润），总股本单位需统一（亿股 vs 万股）
```

| 还原EPS（银行专项） | ¥XX.XX（报表EPS ¥XX.XX） |

##### 第二步：PB 估值锚点校验 + 估值修复溢价

**核心原则**：当 PB 处于历史底部时，即使 EPS 零增长，估值修复本身就能产生显著回报。

```
Step A：计算 PB 历史分位数
        # 使用查询获取的近5年月度PB数据，按升序排列后计算当前PB分位数
        当前PB分位数 = rank(当前PB, PB_历史序列) / len(PB_历史序列) × 100%

Step B：判断是否触发 PB 锚点
        if 当前PB < 0.5 倍：
            触发条件：PB < 0.5 倍（极低估）
        elif 当前PB分位数 < 15%（近5年底部区域）：
            触发条件：PB 历史分位数 < 15%
        else：
            不触发，无需增加溢价

Step C：计算估值修复溢价
        if 触发条件存在：
            # 溢价幅度与PB偏离程度挂钩
            if 当前PB < 0.3 倍：
                修复溢价系数 = 20%（极度低估，溢价上限）
            elif 当前PB < 0.4 倍：
                修复溢价系数 = 15%（显著低估）
            elif 当前PB < 0.5 倍 或 PB分位数 < 15%：
                修复溢价系数 = 10%（底部区域）

            修复溢价 = DDM安全买入价 × 修复溢价系数
            PB调整后安全买入价 = DDM安全买入价 + 修复溢价
            说明："PB处于历史底部（{当前PB}倍，分位数{XX%}），即使EPS零增长，
                   PB从{X}倍回归均值{X}倍的估值修复空间约{XX%}，
                   故在DDM结果基础上叠加{XX}%估值修复溢价"
        else：
            PB调整后安全买入价 = DDM安全买入价
```

**报告标注**：

```
| 指标 | 数值 | 说明 |
|------|------|------|
| 当前PB | X.XX倍 | |
| 近5年PB均值 | X.XX倍 | 历史中枢 |
| 近5年PB区间 | X.XX ~ X.XX倍 | 历史极值 |
| PB分位数 | XX% | 历史底部/中位/高位 |
| PB锚点触发 | ✅/❌ | {触发条件} |
| 估值修复溢价 | +{XX}% | {说明} |
| PB调整后安全买入价 | ¥XX.XX | DDM结果 + 估值修复溢价 |
```

##### 第三步：动态 Target Yield（银行类债属性）

**核心原则**：资本充足率远高于监管要求 + 历史分红极稳 → 类债属性更强 → 可降低目标收益率。

```
Step A：评估资本充足率安全边际
        核心一级资本充足率要求 = 5%（最低监管要求）
        实际核心一级资本充足率 = 查询数据

        安全边际 = 实际核心一级资本充足率 - 最低要求
        # 例如：实际 10.5% - 要求 5% = 安全边际 5.5pp

Step B：判断是否满足 Target Yield 下调条件
        满足条件（需同时满足）：
        ① 核心一级资本充足率 > 最低要求 + 4pp（即 > 9%，安全边际充足）
        ② 近5年分红率标准差 < 5pp（分红极稳定）
        ③ 近3年核心一级资本充足率无显著下滑趋势

        if 满足全部条件：
            if 核心一级资本充足率 > 最低要求 + 6pp（即 > 11%）：
                target_yield_bank = 6.0%（类债属性极强）
                说明："资本充足率极度充裕，类债属性突出"
            else：
                target_yield_bank = 6.5-7.0%（稳健下调）
                说明："资本充足率充裕，类债属性较强"
        else：
            target_yield_bank = Phase 2.4 确定的 target_yield（标准处理）

Step C：说明 Target Yield 调整对安全买入价的影响
        # 目标收益率降低 → 安全买入价升高（与Phase 2.4的提高方向相反）
        # Phase 2.7的银行专项下调优先生效，不与Phase 2.4上调重复叠加
```

**报告标注**：

```
| 指标 | 数值 | 说明 |
|------|------|------|
| 核心一级资本充足率 | XX.X%（监管要求≥5%） | 安全边际 {X}pp |
| 近5年分红率标准差 | X.Xpp | 分红稳定性 |
| Target Yield（银行专项） | {X}% | 下调/标准（Phase 2.4={X}%） |
```

##### 第四步：银行股安全买入价最终取值

**核心原则**：银行股的安全买入价 = **DDM结果** 与 **股息率锚定价** 的较高者。

```
Step A：计算股息率锚定价
        股息率锚定价 = (EPS × 分红率) / target_yield_bank
        # 即：以当前股息率达到目标收益率的价格

        # 示例：EPS ¥5.26，分红率 33.9%，target_yield_bank = 6.5%
        # 股息率锚定价 = (5.26 × 33.9%) / 6.5% = ¥27.40

Step B：取两者较高值
        if Phase 2.7 触发了 PB 修复溢价：
            安全买入价 = max(DDM安全买入价 + 修复溢价, 股息率锚定价)
            说明："DDM结果（含PB修复溢价）¥XX > 股息率锚定价¥XX，取较高值"
        else：
            安全买入价 = max(DDM安全买入价, 股息率锚定价)
            说明："DDM结果¥XX {>/<} 股息率锚定价¥XX，取较高值"
```

##### Phase 2.7 执行流程总结

```
Step 1：识别银行股 → Phase 2.5 识别"PB修复+股息"路径
Step 2：获取银行专项数据（B①资本充足率、B②拨备覆盖率、B③近5年PB）
Step 3：坏账拨备还原
        → 超额拨备 → 还原EPS_Bank
        → 不超额 → 使用报表EPS
Step 4：PB锚点校验
        → PB < 0.5倍 或 分位数 < 15% → 叠加10-20%估值修复溢价
        → 否则 → 无溢价
Step 5：动态Target Yield
        → 资本充足率安全边际充足 + 分红极稳 → 下调至6-6.5-7%
        → 否则 → 维持Phase 2.4确定值（两者不叠加，以银行专项为准）
Step 6：最终安全买入价 = max(DDM(含溢价), 股息率锚定价)
Step 7：继续 Phase 3（筛选条件检验）
```

| 业务确定性 | 商业模式简单，10年后仍被需要（主观判断） | 供参考 |

**负债率可控详细判断规则（⚠ 周期性重资产行业适用强化版）**：

```
# 基础规则（非周期性行业）
if 资产负债率 ≤ 60%：
    结果 = ✅（负债率可控）
elif 60% < 资产负债率 ≤ 70%：
    结果 = ⚠️（负债率偏高）
else：
    结果 = ❌（负债率过高）

# 周期性重资产行业强化规则（化工/有色/航运/钢铁/煤炭/建材/造纸/玻璃 + ROE_std ≥ 6pp）
if ROE_std ≥ 6pp and 行业属于上述类别：
    if 资产负债率 > 70%：
        结果 = ⚠️（严审：Phase 2.4 Step 4 红旗已触发）
        说明："周期性重资产行业 + 资产负债率{X}%>70%，高杠杆在行业下行期存在债务危机风险，
               Phase 2.4 已标记红旗，建议审慎评估，建议在 Phase 3 结论中强化风险提示"
    elif 60% < 资产负债率 ≤ 70%：
        结果 = ⚠️（关注）
        说明："周期性重资产行业负债率偏高，周期下行压力较大"
    else：
        结果 = ✅（负债率可控）
```

**⚠ 一票否决触发场景**：

> 当"历史分红稳定"或"现金流健康"出现 ❌ 时，安全买入价结论中注明"⚠️ 有一票否决项，建议回避"。

### Phase 5：输出至 IMA 笔记本

> **目标**：每次生成估值报告后，自动将 Markdown 报告写入 IMA "股票估值"笔记本，形成可检索的个人估值知识库。

#### 5.1 笔记本定位

```
笔记本名称：股票估值
笔记本 folder_id：folder18c42fb8e51fe8c3
笔记 note_id：（每次新建返回新的 note_id）
```

> **如需验证笔记本是否存在**，可调用：
> ```bash
> node -e "const {execSync}=require('child_process');console.log(execSync('node .../ima_api.cjs openapi/note/v1/list_notebook ...').toString())"
> ```
> 但不必每次都验证——若 folder_id 错误，`import_doc` 会返回 `{code:310001}`（笔记本不存在），此时告知用户检查笔记本。

#### 5.2 写入流程

```
生成 Markdown 报告（Phase 4）
    ↓
UTF-8 校验（必须，防止乱码）
    ↓
直接写入 IMA notebook（import_doc API）
    ↓
记录 note_id 到报告头部注释（如已有则更新）
```

#### 5.3 完整写入脚本

> ⚠️ **重要**：不要通过 shell 调用 `ima_api.cjs`（shell 会将 Markdown 中的 `(1-Payout)` 等括号误判为子shell，导致转义错误）。
> 必须将内容写入临时 JS 文件，再用 Node.js 直接执行。

**写入脚本模板**（保存为 `/tmp/ima_write_report.js`）：

```javascript
#!/usr/bin/env node
/**
 * 写入估值报告到 IMA 笔记本
 * 用法：node /tmp/ima_write_report.js "报告内容（Markdown）" "笔记本folder_id"
 */
const fs = require('fs');
const https = require('https');

const clientId = fs.readFileSync('/Users/yan/.config/ima/client_id', 'utf8').trim();
const apiKey = fs.readFileSync('/Users/yan/.config/ima/api_key', 'utf8').trim();
const content = process.argv[2] || fs.readFileSync('/tmp/report_body.md', 'utf8');
const folderId = process.argv[3] || 'folder18c42fb8e51fe8c3';

const body = { content_format: 1, content, folder_id: folderId };

async function apiCall(path, body) {
  const res = await fetch(`https://ima.qq.com/${path}`, {
    method: 'POST',
    headers: {
      'ima-openapi-clientid': clientId,
      'ima-openapi-apikey': apiKey,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(body),
  });
  const text = await res.text();
  const json = JSON.parse(text);
  if (json.code !== 0) throw new Error(`IMA API error: ${json.code} ${json.msg}`);
  return json.data.note_id;
}

async function main() {
  const noteId = await apiCall('openapi/note/v1/import_doc', body);
  console.log('✅ 报告已写入 IMA，note_id:', noteId);
}

main().catch(e => { console.error('❌', e.message); process.exit(1); });
```

#### 5.4 执行方式（两步走）

**Step 1**：将报告内容保存到临时文件
```bash
cp "/path/to/{股票名称}_估值报告.md" /tmp/report_body.md
```

**Step 2**：执行写入脚本
```bash
node /tmp/ima_write_report.js "$(cat /tmp/report_body.md)" "folder18c42fb8e51fe8c3"
```

> **或一行命令（适用于报告路径已知）**：
> ```bash
> node /tmp/ima_write_report.js "$(cat '/path/to/贵州茅台_估值报告.md')" "folder18c42fb8e51fe8c3"
> ```

#### 5.5 UTF-8 校验（强制）

> 在写入前必须校验报告内容为合法 UTF-8：
> ```bash
> node -e "const b=require('fs').readFileSync('/tmp/report_body.md');try{Buffer.from(b,'utf8');console.log('UTF-8 OK')}catch(e){console.error('❌ 非UTF-8:',e.message)}"
> ```

#### 5.6 错误处理

| IMA 返回码 | 含义 | 处理方式 |
|-----------|------|---------|
| `code: 0` | 成功 | 记录 note_id，完成 |
| `code: 310001` | 笔记本不存在 | 告知用户检查 IMA 中的"股票估值"笔记本是否存在 |

| 其他非0码 | API 错误 | 读取 `msg` 字段，向用户展示错误原因 |

## 核心公式速查

```
安全买入价 = [ EPS_curr × (1 + g_adjusted)^10 × Payout ] / target_yield_adjusted
# g_adjusted = g_base 经 Phase 2.4 周期调整后的值（若无调整，则 = g_final）
# target_yield_adjusted = 8%（稳定）/ 9-10%（波动）/ 10-12%（强周期），由 Phase 2.4 ROE_std 确定

g_base   = ROE_used × (1 - Payout_5y)
# ROE_used = 均值ROE（标准）或 min(近5年ROE)（ROE_std ≥ 6pp时）

g_final 根据 ROE 档位截断：
  ROE > 25%           → min(g_base, 8%)
  15% ≤ ROE ≤ 25%     → min(g_base, 5%)
  ROE < 15%           → min(g_base, 3%)

动态目标收益率（Phase 2.4）：
  ROE_std < 3pp       → target_yield = 8%
  ROE_std 3-6pp       → target_yield = 9-10%
  ROE_std ≥ 6pp       → target_yield = 10-12%

g_adjusted = g_final × 周期折扣系数（Phase 2.4 Step 3）：
  极度过热            → × 0.5
  偏高估              → × 0.7
  正常                → × 1.0
```

### 招商银行（600036.SH）— 回测时点 2023 年 11 月底

**回测背景**：2023 年 11 月底，招商银行股价 ¥23.81（PB ≈ 0.55 倍），市场情绪悲观（LPR 多次下调、经济复苏不及预期）。

| 项目 | 回测输入（2023-11 视角） | 实际结果（2026-04） |
|------|------------------------|-------------------|
| 安全买入价 | ¥36.31（EPS ¥5.26，ROE 16.1%，分红率 33.9%） | 股价 ¥39.60，涨幅 66.3% |
| 安全边际 | +52.5%，🟢 明显低估 | 低估判断正确 ✅ |
| EPS 预期 | 10 年后 EPS ¥8.57（假设 5% 年增长） | 2025 年 EPS ¥5.70，近乎零增长 ⚠️ |
| ROE 预期 | 5 年均 16.1%，但 2022 年已降至 14.49% | 2025 年 ROE 13.44%，持续下行 ⚠️ |
| 分红率 | 5 年均 33.9% | 2025 年升至 35.4% ✅ |

**核心教训（已融入模型指导规则）：**

1. **安全买入价是有效的"价值锚"**：低估时买入的判断正确，即使增长假设不准确，安全边际仍提供了足够的保护
2. **模型对银行股的回报路径估计不完整**：实际上涨驱动力是**估值修复（PB 0.55→0.91）+ 高股息率（8.4%）**，而非 EPS 增长。内生增长模型假设 EPS 持续增长，但银行股可以通过**估值修复**和**股息收益**实现回报
3. **g_final 可信度风险会兑现**：回测时已标记 ROE 下行风险和净息差收窄，2025 年 ROE 降至 13.44% 确认进入 3% 档位。低可信度（🟡 中可信）的警告应引起足够重视
4. **银行股需补充分析维度**：除模型安全价外，应额外关注 **PB 估值位置**（是否处于历史极端低位）和 **股息率**（是否已接近或超过目标收益率），这两个维度对银行股同样重要

### 贵州茅台（600519.SH）— 回测时点 2020 年 11 月底

**回测背景**：2020 年 11 月底，贵州茅台股价 ¥1,499.70（PE ≈ 45.7 倍），处于历史极高估值水平，受益于疫情后流动性宽松和白酒需求恢复。

| 项目 | 回测输入（2020-11 视角） | 实际结果（2026-04） |
|------|------------------------|-------------------|
| 安全买入价 | ¥452.85（EPS ¥32.80，ROE 30.2%，分红率 51.2%） | 股价 ¥1,405.00，5.4 年略跌 6.3% |
| 安全边际 | -231.2%，🔴 明显高估 | 高估判断长期正确 ✅（5 年零收益） |

**核心教训（已融入模型指导规则）：**

1. **高估判断在长期得到验证**：¥1,499.70 买入后 5.4 年几乎零收益（-6.3%），高估确实损害了投资回报。EPS 从 ¥32.80 翻倍至 ¥65.66（+100%），但 PE 从 45.7 降至 21.3（-53%），估值消化完全抵消了业绩增长
2. **高估 ≠ 立即下跌**：茅台在 2020 年底至 2021 年 2 月继续上涨至 ¥2,100+，模型无法预测市场情绪的延续时间。但模型的价值在于**纪律性**——避免高估追涨
3. **品牌溢价突破模型框架**：茅台拥有"类货币"和"社交货币"属性，市场长期给予远超模型安全价的估值溢价。对于品牌护城河极深的优质公司，安全买入价更像**"价值底线"**而非**"合理价格"**
4. **分红率提升超预期**：茅台 2024-2026 年承诺分红率不低于 75%，远超历史均值 51%。分红提升部分对冲了估值下跌，但不足以在高估值下产生正回报
5. **极端高估案例的警示**：PE 45+ 倍的价格意味着即使 EPS 未来 10 年以 8% 增长、分红率维持 51%，也仅能实现约 8% 的目标收益率。一旦增速不及预期或估值收缩，回报将显著低于目标。这是模型对极端高估最有效的警示场景

- 数据质量决定结果质量，分红数据缺失时需明确说明
- **银行股专属修正引擎（Phase 2.7）**：对银行股，Phase 2.7 提供三项专属修正：①坏账拨备还原（识别藏利润行为）；②PB锚点 + 估值修复溢价（PB<0.5倍时叠加10-20%溢价）；③动态Target Yield（资本充足率充裕时分红极稳时可下调至6-7%，反映类债属性）。最终安全买入价 = max(DDM(含溢价), 股息率锚定价)。三项修正联动，确保银行股安全买入价反映"PB修复+股息"双驱动特性。
- **g取值必须结合周期判断**：用"近5年EPS CAGR"前瞻"未来10年增速"，本质上是行业周期判断。若当前处于周期高点（近5年增速年化30%），均值回归压力极大，g要往下调；若处于周期调整末期（近5年增速低甚至负），均值回归向上弹性大，g可以上修。调整的是g_base截断前的原始值，调整后仍需ROE档位截断。
- **顶级消费品牌享有结构性估值溢价**：ROE 持续 >25%、品牌护城河极深、现金流优异的公司，市场长期给予 PE 25-35 倍的估值溢价是合理的，不应简单以模型安全价判断"高估"
- **财务洗澡后必须使用还原EPS**：当检测到会计差错更正（调整幅度>30%）或董事长被查后业绩暴雷，必须用「近3年平均经营现金流÷总股本」作为还原EPS重新计算安全买入价，与报表EPS结果并列对比，取较低者作为保守估值参考
- **强周期行业（钢铁/煤炭/有色）强烈不建议使用此模型**：Phase 2.4 的 ROE_std 预警是最后防线，但即使触发后用最低ROE计算，安全价在周期下行期仍可能严重高估。强周期股应改用 PB-ROE 框架（净资产为锚）或 DCF（以产能周期为基础），本模型仅作"价值下限的粗略参考"
- **成长股（医药/科技等低分红）强烈不建议使用此模型**：模型假设分红增长，但成长股不分红或分红极少，安全价计算结果严重低估实际价值。必须配合 PEG（PE ÷ 预期增速）综合判断；PEG < 1 是被低估的信号，PEG > 2 确认高估
- **高回购公司（腾讯等）需用总资本回报率辅助判断**：纯现金分红率无法反映真实的资本回报水平；将回购注销金额计入总资本回报率，才能正确评估此类公司的"股东价值创造"能力
- **PE处于历史极端高位时，模型可能产生误导**：即使g和dividend看似合理，若当前PE处于历史90%+分位，估值收缩风险极大（茅台2020年PE 45倍教训）。Phase 2.6的PE分位数检测是对此类风险的第一道防线
- **行业结构性断裂时，历史ROE完全失效**：房地产（政策重构）、互联网（反垄断）、教培（行业清零）等案例表明，当外生冲击改变了行业竞争逻辑时，模型输入已无意义。必须扫描Phase 2.6的5类外生断裂信号，若存在则g直接清零，安全价仅供参考下限
- **财务洗澡检测已自动化**：Phase 1的偏差熔断机制（偏差率>30%自动双版本输出）是财务风险的强制兜底；偏差率15-30%时还原EPS仅供参考；即使偏差率<15%不触发熔断，仍应在⑥利润质量评分中诚实反映OCF匹配度
- **周期性重资产行业（化工/有色/航运/钢铁/煤炭等）需强化三项保护**（Phase 2.4 专项）：①ROE取值改为min(5年均值, 最近一年, 行业中位数)，而非仅用历史最低值；②目标收益率强制设为10-12%；③资产负债率>70%触发红旗，Phase 3"负债率可控"降为⚠️严审。此类行业的安全买入价极度保守，模型仅作价值下限参考，建议配合PB-ROE或DCF框架综合判断。
